# 中国剩余定理（CRT）

> 中国剩余定理（Chinese Remainder Theorem, CRT）是数论中最为优美且实用的定理之一。
> 它回答了一个基本问题：**给定一组互质的模数，能否从各个模数下的余数唯一恢复出原数？**
> 答案是肯定的，而且构造是显式的。

## 诞生背景与核心原理

### 《孙子算经》——"物不知数"（约 4-5 世纪，南北朝）

中国剩余定理最早的记载见于南北朝时期的《孙子算经》（作者不详，约公元 400-500 年）。书中第 26 题（卷下）记载了著名的"物不知数"问题：

> **原文**：今有物不知其数，三三数之剩二，五五数之剩三，七七数之剩二，问物几何？
>
> **答曰**：二十三。
>
> **术曰**：三三数之剩二，置一百四十；五五数之剩三，置六十三；七七数之剩二，置三十。并之，得二百三十三，以二百一十减之，即得。凡三三数之剩一，则置七十；五五数之剩一，则置二十一；七七数之剩一，则置十五。一百六以上，以一百五减之，即得。

用现代数学语言表述：

```
x ≡ 2 (mod 3)
x ≡ 3 (mod 5)
x ≡ 2 (mod 7)
```

《孙子算经》给出的解法实质上已经包含了 CRT 公式的核心思想：

- **三三数剩二 → 置 140**：70 × 2 = 140，其中 70 ≡ 1 (mod 3) 且 70 ≡ 0 (mod 5) 且 70 ≡ 0 (mod 7)
- **五五数剩三 → 置 63**：21 × 3 = 63，其中 21 ≡ 1 (mod 5) 且被 3、7 整除
- **七七数剩二 → 置 30**：15 × 2 = 30，其中 15 ≡ 1 (mod 7) 且被 3、5 整除

求和得 233，模 105 → x = 23。

> 注意"置七十""置二十一""置十五"——这正是后人总结的**孙子歌诀**的来源。

#### 孙子歌诀（明代程大位《算法统宗》，1592 年）

```
三人同行七十稀（3→70，70 mod 3 = 1）
五树梅花廿一枝（5→21，21 mod 5 = 1）
七子团圆正月半（7→15，15 mod 7 = 1）
除百零五便得知（105 = 3×5×7）
```

这首口诀正是 CRT 构造性证明的完美体现：**先构造每个模数下的"单位解"，再用余数加权求和**。

### 秦九韶《数书九章》——大衍求一术（1247 年，南宋）

南宋数学家**秦九韶**在《数书九章》（1247 年）中将孙子问题推广为一般形式，提出了**大衍总数术**，其中核心算法为**大衍求一术**。

秦九韶的贡献包括：

1. **一般化**：将问题从三个模数推广到任意多个模数
2. **求一术**：给出求解模逆元的系统方法（求一个数，使其模 m 等于 1）——这正是现代**扩展欧几里得算法**的雏形
3. **分类讨论**：处理模数非互质时的约化方法

大衍求一术的核心思想：
> 求乘率 k，使得 k × (M/mᵢ) ≡ 1 (mod mᵢ)

其中"乘率"即现代数学中的**模逆元**。秦九韶使用辗转相除法（欧几里得算法）计算这个乘率，本质上与今天的扩展欧几里得算法完全相同。

### Gauss《算术研究》——现代形式（1801 年）

**卡尔·弗里德里希·高斯**（Carl Friedrich Gauss）在其里程碑式著作《算术研究》（*Disquisitiones Arithmeticae*, 1801 年）中，以完全一般化的形式给出了中国剩余定理并给出了严格证明。

#### Gauss 的定理表述

> **定理（CRT, Gauss 1801）**：若模数 m₁, m₂, ..., mₖ 两两互质，则对于任意给定的余数 a₁, a₂, ..., aₖ，同余方程组：
> ```
> x ≡ a₁ (mod m₁)
> x ≡ a₂ (mod m₂)
> ...
> x ≡ aₖ (mod mₖ)
> ```
> 在模 M = m₁×m₂×...×mₖ 下存在**唯一解**。

Gauss 提供了**构造性证明**，即直接给出解的公式。

### CRT 构造公式的数学推导

#### 核心思想：化归为单位问题

欲求 x 满足所有方程，先求 **k 个特殊数 eᵢ**，使得：

```
eᵢ ≡ 1 (mod mᵢ)
eᵢ ≡ 0 (mod mⱼ)  (j ≠ i)
```

则原问题的解为：**x ≡ a₁·e₁ + a₂·e₂ + ... + aₖ·eₖ (mod M)**

#### 如何构造 eᵢ？

由于需要 eᵢ ≡ 0 (mod mⱼ) 对所有 j ≠ i 成立，eᵢ 必须是所有 mⱼ (j ≠ i) 的倍数，即：

```
eᵢ = Mᵢ × kᵢ，其中 Mᵢ = M / mᵢ = Π_{j≠i} mⱼ
```

现在需要 eᵢ ≡ 1 (mod mᵢ)，即：

```
Mᵢ × kᵢ ≡ 1 (mod mᵢ)
```

这等价于求 **kᵢ 为 Mᵢ 在模 mᵢ 下的逆元**，记作 kᵢ = Mᵢ⁻¹ (mod mᵢ)。

> **为什么逆元存在？** 因为模数两两互质 → gcd(Mᵢ, mᵢ) = 1 → Mᵢ 在模 mᵢ 下有逆元。

因此：

```
eᵢ = Mᵢ × Mᵢ⁻¹ (mod mᵢ)
```

#### 最终公式

```
x ≡ Σ aᵢ × Mᵢ × inv(Mᵢ) (mod M)，其中 M = Π mᵢ，inv(Mᵢ) 在模 mᵢ 下计算
```

**构造的直观理解**：
- 第 i 项 `aᵢ × Mᵢ × inv(Mᵢ)` 仅对模 mᵢ 的贡献为 aᵢ（因为 Mᵢ × inv(Mᵢ) ≡ 1 mod mᵢ），对模 mⱼ (j≠i) 的贡献为 0（因为 Mᵢ 是 mⱼ 的倍数）
- 求和后，每个模数下都恰好得到对应的余数

#### 唯一性证明

假设 x 和 x' 都是解，则对每个 i：

```
x ≡ aᵢ (mod mᵢ) 且 x' ≡ aᵢ (mod mᵢ)
→ x - x' ≡ 0 (mod mᵢ)
```

所以 x - x' 是每个 mᵢ 的倍数。由于 mᵢ 两两互质，x - x' 是 M = Π mᵢ 的倍数，即 x ≡ x' (mod M)。

### 扩展 CRT（ExCRT）的推导

当模数**不互质**时，不能直接使用 CRT 公式（因为 Mᵢ 和 mᵢ 的 gcd 非 1，逆元不存在）。
但我们可以**两两合并**，这就是扩展中国剩余定理（ExCRT）。

#### 合并两个方程的推导

对于两个方程：

```
x ≡ r₁ (mod m₁)
x ≡ r₂ (mod m₂)
```

由第一个方程设 x = r₁ + m₁·t，代入第二个方程：

```
r₁ + m₁·t ≡ r₂ (mod m₂)
→ m₁·t ≡ r₂ - r₁ (mod m₂)
```

这是一个**线性同余方程**。设 d = gcd(m₁, m₂)：

- **可解条件**：d | (r₂ - r₁)。若 d ∤ (r₂ - r₁)，则无解
- 若可解，两边同除以 d：`(m₁/d)·t ≡ (r₂-r₁)/d (mod m₂/d)`
- 由于 m₁/d 与 m₂/d 互质，可求逆元：`t ≡ ((r₂-r₁)/d) × (m₁/d)⁻¹ (mod m₂/d)`
- 取特解 t₀ = p × (r₂-r₁)/d（其中 p 满足 m₁·p + m₂·q = d，来自扩展欧几里得）
- 则 `x ≡ r₁ + m₁·t₀ (mod lcm(m₁, m₂))`

#### 合并后的新方程

```
新模数：m = lcm(m₁, m₂) = m₁ / d × m₂
新余数：r = (r₁ + m₁ × t₀) mod m
```

不断合并直到只剩一个方程，或某步无解。

## 核心问题与适用边界

### 为什么 CRT 要求模数两两互质？

**根本原因**：求逆元时需要 gcd(Mᵢ, mᵢ) = 1。

回顾构造公式 eᵢ = Mᵢ × kᵢ，其中 kᵢ 满足 Mᵢ × kᵢ ≡ 1 (mod mᵢ)。

根据数论基本定理：
> a 在模 m 下有逆元 **当且仅当** gcd(a, m) = 1。

若模数非互质，例如 m₁ = 6, m₂ = 10，则 M₁ = 10，需要解：
```
10 × k₁ ≡ 1 (mod 6)
→ gcd(10, 6) = 2 ≠ 1 → 逆元不存在！
```

实际上，此时 eᵢ 恒为偶数，不可能 ≡ 1 (mod 6)。因此 CRT 公式直接失效。

#### 反例说明

```
x ≡ 0 (mod 4)
x ≡ 0 (mod 6)
```

用 CRT 公式强行计算：M = 24, M₁ = 6, 求 6 模 4 的逆元——gcd(6,4)=2，无解。
但实际方程组的解是 x ≡ 0, 12, 24,...（模 12），解确实存在！

**结论**：非互质时解**可能**存在，但 CRT 标准公式失效，需用 ExCRT。

### 非互质时的整除性条件（ExCRT）

给定两方程：

```
x ≡ r₁ (mod m₁)
x ≡ r₂ (mod m₂)
```

设 d = gcd(m₁, m₂)，**必要条件**：d | (r₂ - r₁)

**证明**：若解 x 存在，则 x = r₁ + m₁·s = r₂ + m₂·t，故 r₂ - r₁ = m₁·s - m₂·t，右端是 d 的倍数，⇒ d | (r₂ - r₁)。

**充分性**：若 d | (r₂ - r₁)，则一定存在解（由扩展欧几里得给出构造）。

> 对 k 个方程的 ExCRT，两两合并过程中每一步都需要检查这个条件。**某一步不满足即整体无解**。

### 大数溢出问题

#### 问题描述

CRT 需要计算 M = Π mᵢ。如果模数较大（例如 10 个约 10⁹ 的数），M 可达 10⁹⁰，远超：
- `long`（64 位）：≈ 9.22 × 10¹⁸
- `int`（32 位）：≈ 2.15 × 10⁹

#### 何时需要 BigInteger

| 场景 | 模数大小 | 乘积范围 | 适用类型 |
|------|---------|---------|---------|
| 算法竞赛简单题 | mᵢ ≤ 10⁶, k ≤ 10 | ≤ 10⁶⁰ | 必须 BigInteger |
| 算法竞赛中档题 | mᵢ ≤ 10⁹, k ≤ 5 | ≤ 10⁴⁵ | 必须 BigInteger |
| RSA-CRT | p,q ≈ 2¹⁰²⁴ | ≈ 2²⁰⁴⁸ | 必须 BigInteger |
| 工程应用 | 自定义小模数 ≤ 10⁹ | 乘积可控 | long 或 BigInteger |
| Goldman 预计算 | 固定小模数集 | ≤ 10¹⁸ | long 安全 |

**黄金法则**：只要 k ≥ 3 且模数 ≥ 10⁵，就用 BigInteger。**永远不要假设乘积不溢出**。

#### mulMod 技巧

即使在 long 范围内，计算 `result = (result + a_i * (Mi % M) * inv) % M` 时中间结果也可能溢出 long。使用 **mulMod**（快速乘模，类似快速幂）：

```java
// O(log b) 的快速乘模，避免溢出
long mulMod(long a, long b, long mod) {
    long res = 0;
    a %= mod;
    while (b > 0) {
        if ((b & 1) == 1) res = (res + a) % mod;
        a = (a << 1) % mod;
        b >>= 1;
    }
    return res;
}
```

### 方程数量 k 对复杂度的影响

| 算法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 标准 CRT（互质） | O(k · log max(mᵢ)) | O(k) | 每个方程求一次扩展欧几里得 |
| ExCRT（非互质） | O(k · log max(mᵢ)) | O(1) | 两两合并，总数 k-1 次合并 |
| Garner 算法 | O(k²) | O(k) | 双重循环，适合模数固定多次求解 |
| Goldman 预计算 | O(k · log max(mᵢ)) 预计算 + O(k) 每次查询 | O(k) | 一次预计算，多次快速查询 |

> 所有算法的瓶颈都在**扩展欧几里得**（O(log max(mᵢ))）上。k 本身一般较小（竞赛中 k ≤ 10~100），因此总体开销可控。

## 高效实现与关键优化

### Garner 算法

#### 算法思想

Garner 算法是另一种 CRT 实现方式，采用**混合进制表示**：将解 x 表示为：

```
x = x₀ + x₁·m₁ + x₂·(m₁·m₂) + ... + xₖ₋₁·(m₁·m₂·...·mₖ₋₁)
```

其中 0 ≤ xᵢ < mᵢ₊₁。这种表示法避免了在中间步骤处理巨大的 M。

#### 算法步骤

1. 设 x = a₁（第一余数），M_cum = 1
2. 对 i = 2 到 k：
   - 求 t 使得 x + M_cum·t ≡ aᵢ (mod mᵢ)
   - t = (aᵢ - x) × (M_cum)⁻¹ (mod mᵢ)
   - x = x + M_cum × t
   - M_cum = M_cum × mᵢ

#### Java 实现

```java
/**
 * Garner 算法（模数互质）
 * 特点：避免了中间的大整数乘法，适合模数固定、余数变化多次查询的场景
 * 时间复杂度 O(k²)（主要是因为每次需要求逆元，可用预计算优化为 O(k)）
 */
long garner(long[] remainders, long[] moduli) {
    int k = moduli.length;
    long x = remainders[0] % moduli[0];
    long M = 1;  // 累积模数乘积

    for (int i = 1; i < k; i++) {
        M *= moduli[i - 1];
        // 解 x + M·t ≡ aᵢ (mod mᵢ)
        // → M·t ≡ aᵢ - x (mod mᵢ)
        long diff = (remainders[i] - x) % moduli[i];
        if (diff < 0) diff += moduli[i];
        long inv = modInverse(M % moduli[i], moduli[i]); // gcd(M, mᵢ) = 1（互质保证）
        long t = (diff * inv) % moduli[i];
        x = x + M * t;
    }
    return x % (M * moduli[k - 1]);
}
```

#### 与标准 CRT 对比

| 特性 | 标准 CRT | Garner |
|------|---------|--------|
| 公式 | x = Σ aᵢ·Mᵢ·inv(Mᵢ) | x = 混合基表示 |
| 中间值 | 需要计算 M = Π mᵢ（可能极大） | 逐步累积，无巨大乘积 |
| 多次查询 | 每次需重新计算所有 Mᵢ 和逆元 | 可预计算逆元，O(k) 每次 |
| 适用场景 | 单次求解，模数较小 | 模数固定，多次查询 |

#### 预计算优化的 Garner

```java
class GarnerPrecomputed {
    long[][] invTable; // invTable[i][j] = (mᵢ)⁻¹ mod mⱼ (i < j)
    long[] moduli;

    GarnerPrecomputed(long[] moduli) {
        this.moduli = moduli;
        int k = moduli.length;
        invTable = new long[k][k];

        for (int i = 0; i < k; i++) {
            for (int j = i + 1; j < k; j++) {
                // 需 mᵢ 与 mⱼ 互质
                invTable[i][j] = modInverse(moduli[i] % moduli[j], moduli[j]);
            }
        }
    }

    long solve(long[] remainders) {
        int k = moduli.length;
        long[] coef = new long[k];
        coef[0] = remainders[0] % moduli[0];

        for (int i = 1; i < k; i++) {
            long sum = coef[0];
            long prod = 1;
            for (int j = 1; j < i; j++) {
                prod = (prod * moduli[j - 1]) % moduli[i];
                sum = (sum + prod * coef[j]) % moduli[i];
            }
            prod = (prod * moduli[i - 1]) % moduli[i];
            long diff = (remainders[i] - sum) % moduli[i];
            if (diff < 0) diff += moduli[i];
            coef[i] = (diff * modInverse(prod, moduli[i])) % moduli[i];
        }

        long x = coef[0];
        long M = 1;
        for (int i = 1; i < k; i++) {
            M *= moduli[i - 1];
            x += M * coef[i];
        }
        return x;
    }
}
```

### 幂等预处理（Goldman 方法）

当**模数固定**且需要多次用不同余数求解时，可预计算 `Mᵢ × Mᵢ⁻¹ (mod M)`，每次查询 O(k)。

```java
class CRTPrecomputed {
    long[] moduli;
    long[] units;   // 预计算的 Mᵢ × inv(Mᵢ) mod M
    long M;

    /**
     * 预计算（O(k·log max(mᵢ))）
     * 模数固定不变，可复用该对象求解多组余数
     */
    CRTPrecomputed(long[] moduli) {
        this.moduli = moduli;
        M = 1;
        for (long m : moduli) M *= m;

        units = new long[moduli.length];
        for (int i = 0; i < moduli.length; i++) {
            long Mi = M / moduli[i];
            long inv = modInverse(Mi % moduli[i], moduli[i]);
            units[i] = mulMod(Mi % M, inv % M, M);
        }
    }

    /**
     * 查询求解（O(k)）
     * x = Σ aᵢ × unitsᵢ (mod M)
     */
    long solve(long[] remainders) {
        long x = 0;
        for (int i = 0; i < moduli.length; i++) {
            x = (x + mulMod(remainders[i] % M, units[i], M)) % M;
        }
        return (x % M + M) % M;
    }
}
```

### mulMod 防溢出乘法

```java
/**
 * 快速乘模（Russian Peasant Multiplication）
 * 在模数 M 可达 10¹⁸ 甚至更大的情况下，直接 a*b%M 可能溢出 64 位
 * 使用类似快速幂的"二进制展开"法，将乘法拆解为加法
 *
 * 原理：a × b = a × (b₀·2⁰ + b₁·2¹ + ... + b₆₃·2⁶³)
 *         = a·b₀ + 2·(a·b₁ + 2·(...))
 * 每次只做加法，加法用 % mod 保证不溢出
 */
long mulMod(long a, long b, long mod) {
    long res = 0;
    a %= mod;
    b %= mod;
    while (b > 0) {
        if ((b & 1) == 1) res = (res + a) % mod;
        a = (a << 1) % mod;
        b >>= 1;
    }
    return res;
}

/**
 * 更快的 mulMod（当 mod < 2⁶² 时可用）
 * 利用 Java 的 128 位整数乘法（通过溢出截断 + 补偿）
 * 比循环版快约 10 倍
 */
long mulModFast(long a, long b, long mod) {
    long res = 0;
    while (b > 0) {
        if ((b & 1) == 1) {
            res += a;
            if (res >= mod) res -= mod;
        }
        a <<= 1;
        if (a >= mod) a -= mod;
        b >>= 1;
    }
    return res;
}

/**
 * 使用 BigInteger 的 mulMod（当 mod 为 64 位任意值时最安全）
 */
long mulModSafe(long a, long b, long mod) {
    return BigInteger.valueOf(a)
        .multiply(BigInteger.valueOf(b))
        .mod(BigInteger.valueOf(mod))
        .longValue();
}
```

### ExCRT 中带符号 GCD 处理

在 ExCRT 合并中需要特别注意**负数的处理**和**带符号的扩展欧几里得**。

```java
/**
 * 扩展欧几里得算法
 * 返回 [g, x, y]，满足 a·x + b·y = g = gcd(a, b)
 *
 * 注意：x 可为负，使用时需要取模修正
 */
long[] exgcd(long a, long b) {
    if (b == 0) return new long[]{a, 1, 0};
    long[] res = exgcd(b, a % b);
    long g = res[0], x1 = res[1], y1 = res[2];
    return new long[]{g, y1, x1 - (a / b) * y1};
}

/**
 * ExCRT 合并（完整版，正确处理符号）
 */
class Congruence {
    long r; // 余数（0 ≤ r < m）
    long m; // 模数（正数）

    Congruence(long r, long m) {
        this.r = (r % m + m) % m; // 规范化到 [0, m)
        this.m = m;
    }

    // x ≡ r (mod m) 的字符串表示
    public String toString() {
        return "x ≡ " + r + " (mod " + m + ")";
    }
}

/**
 * 合并两个同余方程（核心函数）
 *
 * 推导过程：
 * x = r₁ + m₁·t       (1)
 * r₁ + m₁·t ≡ r₂ (mod m₂)
 * m₁·t ≡ r₂ - r₁ (mod m₂)
 *
 * 设 g = gcd(m₁, m₂)
 * 有解 ⟺ g | (r₂ - r₁)
 * 若可解，用扩展欧几里得求特解 t₀
 *
 * 返回 null 表示无解
 */
Congruence merge(Congruence c1, Congruence c2) {
    long r1 = c1.r, m1 = c1.m;
    long r2 = c2.r, m2 = c2.m;

    long[] ext = exgcd(m1, m2);
    long g = ext[0];          // gcd(m1, m2)
    long p = ext[1];          // 系数，满足 m1·p + m2·q = g

    long diff = r2 - r1;

    // === 无解判定 ===
    // 若 diff 不能被 g 整除，方程组无解
    if (diff % g != 0) return null;

    // === 求特解 t₀ ===
    // m₁·p + m₂·q = g → m₁·p ≡ g (mod m₂)
    // 两边乘以 diff/g：m₁·(p·diff/g) ≡ diff (mod m₂)
    // 故 t₀ = p · diff / g (mod m₂/g)
    long m2g = m2 / g;
    long t = (diff / g % m2g) * (p % m2g) % m2g;
    if (t < 0) t += m2g;

    // === 合并 ===
    // x = r₁ + m₁·t₀ 是特解
    // 新模数 = lcm(m₁, m₂) = m₁/g × m₂
    // 注意：先除再乘，防止溢出
    long newM = m1 / g * m2; // lcm
    long newR = (r1 + mulMod(m1 % newM, t % newM, newM)) % newM;
    if (newR < 0) newR += newM;

    return new Congruence(newR, newM);
}
```

### Java BigInteger 版的完整实现

当模数乘积超 64 位时，必须使用 BigInteger。

```java
import java.math.BigInteger;

class CRTHelper {

    // ==================== BigInteger 扩展欧几里得 ====================

    /**
     * 扩展欧几里得（BigInteger 版）
     * 返回 [g, x, y]，满足 a·x + b·y = g = gcd(a, b)
     */
    static BigInteger[] exgcd(BigInteger a, BigInteger b) {
        if (b.equals(BigInteger.ZERO)) {
            return new BigInteger[]{a, BigInteger.ONE, BigInteger.ZERO};
        }
        BigInteger[] res = exgcd(b, a.mod(b));
        BigInteger g = res[0], x1 = res[1], y1 = res[2];
        return new BigInteger[]{g, y1, x1.subtract(a.divide(b).multiply(y1))};
    }

    /**
     * BigInteger 模逆元（相当于 modInverse）
     */
    static BigInteger modInv(BigInteger a, BigInteger m) {
        BigInteger[] ext = exgcd(a, m);
        if (!ext[0].equals(BigInteger.ONE)) return null; // 无逆元
        return ext[1].mod(m);
    }

    // ==================== CRT（互质）= ====================

    /**
     * CRT：模数两两互质，用公式直接求解
     * 所有参数为 BigInteger，无溢出风险
     *
     * x = Σ aᵢ × Mᵢ × inv(Mᵢ) mod M
     */
    static BigInteger crt(BigInteger[] remainders, BigInteger[] moduli) {
        int k = moduli.length;
        BigInteger M = BigInteger.ONE;
        for (BigInteger m : moduli) M = M.multiply(m);

        BigInteger result = BigInteger.ZERO;
        for (int i = 0; i < k; i++) {
            BigInteger Mi = M.divide(moduli[i]);
            BigInteger inv = Mi.mod(moduli[i]).modInverse(moduli[i]);
            // 注意：modInverse 要求 gcd(Mi, moduli[i]) = 1（由互质保证）
            BigInteger term = remainders[i].multiply(Mi).multiply(inv);
            result = result.add(term).mod(M);
        }
        return result;
    }

    // ==================== ExCRT（非互质）= ====================

    /**
     * ExCRT 合并一步（BigInteger 版）
     */
    static CongruenceBI merge(CongruenceBI c1, CongruenceBI c2) {
        BigInteger r1 = c1.r, m1 = c1.m;
        BigInteger r2 = c2.r, m2 = c2.m;

        BigInteger[] ext = exgcd(m1, m2);
        BigInteger g = ext[0], p = ext[1]; // m1·p + m2·q = g

        BigInteger diff = r2.subtract(r1);

        // 无解判定
        if (!diff.mod(g).equals(BigInteger.ZERO)) return null;

        BigInteger m2g = m2.divide(g);
        BigInteger t = diff.divide(g).mod(m2g).multiply(p.mod(m2g)).mod(m2g);
        if (t.signum() < 0) t = t.add(m2g);

        // lcm = m1/g * m2（先除再乘，防溢出）
        BigInteger newM = m1.divide(g).multiply(m2);
        BigInteger newR = r1.add(m1.multiply(t)).mod(newM);
        if (newR.signum() < 0) newR = newR.add(newM);

        return new CongruenceBI(newR, newM);
    }

    /**
     * ExCRT（BigInteger 版）
     */
    static BigInteger excrt(BigInteger[] remainders, BigInteger[] moduli) {
        CongruenceBI cur = new CongruenceBI(remainders[0], moduli[0]);
        for (int i = 1; i < moduli.length; i++) {
            cur = merge(cur, new CongruenceBI(remainders[i], moduli[i]));
            if (cur == null) return null; // 无解
        }
        return cur.r;
    }

    // 辅助类
    static class CongruenceBI {
        BigInteger r, m;
        CongruenceBI(BigInteger r, BigInteger m) {
            this.r = r.mod(m);
            if (this.r.signum() < 0) this.r = this.r.add(m);
            this.m = m;
        }
    }
}
```

### 快速确定无解条件的推导

在 ExCRT 合并过程中，检查`diff % g == 0`即可判断无解。这里给出完整的代数推导。

#### 问题定位

给定两个同余方程：
```
x ≡ r₁ (mod m₁)
x ≡ r₂ (mod m₂)
```

存在整数 k₁, k₂ 使得：
```
x = r₁ + m₁·k₁
x = r₂ + m₂·k₂
```

相减得：**m₁·k₁ - m₂·k₂ = r₂ - r₁ = diff**

#### 定理

> **Bezout 定理**：线性丢番图方程 m₁·k₁ + m₂·(-k₂) = diff 有解 ⟺ gcd(m₁, m₂) | diff。

**证明**：
- (⇒) 左端 m₁·k₁ - m₂·k₂ 显然是 gcd(m₁, m₂) 的倍数，故 diff 也是
- (⇐) 若 g = gcd(m₁, m₂) | diff，设 g·t = diff。
  由扩展欧几里得存在 s, w 使 m₁·s + m₂·w = g，
  两边乘 t 得 m₁·(s·t) + m₂·(w·t) = diff，
  取 k₁ = s·t, k₂ = -w·t 即得解

#### 判定流程

```
对每一对合并：
  Step 1: 计算 g = gcd(m₁, m₂)
  Step 2: 计算 diff = r₂ - r₁
  Step 3: 若 diff % g ≠ 0 → 无解（立即终止）
  Step 4: 否则继续合并

最终：
  - 全部合并成功 → 解为 x ≡ r_final (mod lcm(m₁,...,mₖ))
  - 任何一步失败  → 方程组无解
```

> **注意**：即使所有方程各自两两之间有解，也不能保证整体有解（因为 ExCRT 是顺序合并，前一步的结果会影响后续无解判定）。但 ExCRT 的顺序合并算法是充要的——只要最终合并成功，解就是正确的。

### 4a. 经典物不知数 —— 直接 CRT

**题目**：今有物不知其数，三三数之剩二，五五数之剩三，七七数之剩二。问物几何？

**分析**：模数 3, 5, 7 两两互质，直接用标准 CRT 公式。

**推导**：
```
M = 3 × 5 × 7 = 105
M₁ = 35,  inv(M₁ mod 3) = inv(2 mod 3) = 2,  e₁ = 35×2 = 70
M₂ = 21,  inv(21 mod 5) = inv(1 mod 5) = 1,  e₂ = 21×1 = 21
M₃ = 15,  inv(15 mod 7) = inv(1 mod 7) = 1,  e₃ = 15×1 = 15
x = 2×70 + 3×21 + 2×15 = 140 + 63 + 30 = 233
x = 233 mod 105 = 23
```

**Java 代码**：

```java
public class ClassicSunzi {
    public static void main(String[] args) {
        long[] remainders = {2, 3, 2}; // 余数
        long[] moduli = {3, 5, 7};     // 模数

        long x = crt(remainders, moduli);
        System.out.println("x = " + x); // 输出: x = 23

        // 验证
        System.out.println("23 % 3 = " + (23 % 3)); // 2
        System.out.println("23 % 5 = " + (23 % 5)); // 3
        System.out.println("23 % 7 = " + (23 % 7)); // 2
    }

    // 标准 CRT（mulMod 防溢出）
    static long crt(long[] a, long[] m) {
        int k = m.length;
        long M = 1;
        for (long mi : m) M *= mi;

        long result = 0;
        for (int i = 0; i < k; i++) {
            long Mi = M / m[i];
            long inv = modInverse(Mi % m[i], m[i]);
            // 使用 mulMod 防止中间结果溢出
            long term = mulMod(mulMod(a[i] % M, Mi % M, M), inv % M, M);
            result = (result + term) % M;
        }
        return (result % M + M) % M;
    }

    static long modInverse(long a, long m) {
        long[] res = exgcd(a, m);
        if (res[0] != 1) return -1; // 无逆元
        return (res[1] % m + m) % m;
    }

    static long[] exgcd(long a, long b) {
        if (b == 0) return new long[]{a, 1, 0};
        long[] res = exgcd(b, a % b);
        return new long[]{res[0], res[2], res[1] - (a / b) * res[2]};
    }

    static long mulMod(long a, long b, long mod) {
        long res = 0;
        a %= mod;
        while (b > 0) {
            if ((b & 1) == 1) res = (res + a) % mod;
            a = (a << 1) % mod;
            b >>= 1;
        }
        return res;
    }
}
```

**复杂度**：O(k·log max(mᵢ)) = O(3·log 7)，常数时间。

**测试用例**：
```
输入: a = [2,3,2],  m = [3,5,7]
输出: 23
验证: 23 % 3 = 2 ✓, 23 % 5 = 3 ✓, 23 % 7 = 2 ✓
```

### 4b. 模不互质方程组 —— ExCRT

**题目**：解同余方程组
```
x ≡ 2 (mod 4)
x ≡ 4 (mod 6)
x ≡ 5 (mod 10)
```

**分析**：模数 4, 6, 10 非互质（gcd(4,6)=2, gcd(6,10)=2），需用 ExCRT 两两合并。

**推导**：

**Step 1：合并第 1、2 方程**
```
m₁=4, m₂=6, g=gcd(4,6)=2
diff = 4-2 = 2, diff % g = 0 ✓

扩展欧几里得：4·p + 6·q = 2 → 4·(-1) + 6·1 = 2 → p = -1
t = diff/g × p mod (m₂/g) = 2/2 × (-1) mod 3 = -1 mod 3 = 2
r' = r₁ + m₁·t = 2 + 4×2 = 10
m' = lcm(4,6) = 12

合并结果：x ≡ 10 (mod 12)
验证：10 % 4 = 2 ✓, 10 % 6 = 4 ✓
```

**Step 2：合并与第 3 方程**
```
m₁=12, m₂=10, g=gcd(12,10)=2
diff = 5-10 = -5, diff % g = (-5) % 2 = 1 ≠ 0 → 无解！
```

**结论**：方程组无解。

**修改题目**：将第三方程改为 x ≡ 3 (mod 10)
```
验证：diff = 3-10 = -7, (-7) % 2 = 1 ≠ 0 → 仍然无解！
```

再改：x ≡ 1 (mod 10)
```
diff = 1-10 = -9, (-9) % 2 = -1, (-1) % 2 = -1 ≠ 0（Java 中 -1 % 2 = -1！）
```

> **重要**：Java 的 `%` 是**余数**而非**模运算**！`(-9) % 2 = -1`，需要用 `((diff % g) + g) % g` 检查。

**修正检查**：

boolean check = ((diff % g) + g) % g == 0;
```

**继续推导**：x ≡ 0 (mod 10)
```
diff = 0-10 = -10, -10 % 2 = 0 ✓

扩展欧几里得：12·p + 10·q = 2 → 12·1 + 10·(-1) = 2 → p = 1
t = diff/g × p mod (m₂/g) = (-10)/2 × 1 mod 5 = -5 mod 5 = 0
r' = 10 + 12×0 = 10
m' = lcm(12,10) = 60

最终解：x ≡ 10 (mod 60)
验证：10 % 4 = 2 ✓, 10 % 6 = 4 ✓, 10 % 10 = 0 ✓

**Java 代码**：

```java
public class ExcrtExample {
    public static void main(String[] args) {
        // 有解的例子
        long[] remainders = {2, 4, 0};
        long[] moduli = {4, 6, 10};

        Long x = excrt(remainders, moduli);
        if (x != null) {
            System.out.println("x ≡ " + x + " (mod " + moduli[0] + ")");
            // 实际解在模 lcm(4,6,10)=60 下为 10
        } else {
            System.out.println("无解");
        }
    }

    static class Congruence {
        long r, m;
        Congruence(long r, long m) {
            this.r = (r % m + m) % m;
            this.m = m;
        }
    }

    static Long excrt(long[] remainders, long[] moduli) {
        Congruence cur = new Congruence(remainders[0], moduli[0]);
        for (int i = 1; i < moduli.length; i++) {
            cur = merge(cur, new Congruence(remainders[i], moduli[i]));
            if (cur == null) return null;
        }
        return cur.r;
    }

    static Congruence merge(Congruence c1, Congruence c2) {
        long r1 = c1.r, m1 = c1.m;
        long r2 = c2.r, m2 = c2.m;

        long[] ext = exgcd(m1, m2);
        long g = ext[0], p = ext[1];

        long diff = r2 - r1;

        // 无解判定（注意负数处理）
        if ((diff % g + g) % g != 0) return null;

        long m2g = m2 / g;
        long t = ((diff / g) % m2g) * (p % m2g) % m2g;
        if (t < 0) t += m2g;

        long newM = m1 / g * m2;
        long newR = (r1 + mulMod(m1 % newM, t % newM, newM)) % newM;
        if (newR < 0) newR += newM;

        return new Congruence(newR, newM);
    }

    static long[] exgcd(long a, long b) {
        if (b == 0) return new long[]{a, 1, 0};
        long[] res = exgcd(b, a % b);
        return new long[]{res[0], res[2], res[1] - (a / b) * res[2]};
    }

    static long mulMod(long a, long b, long mod) {
        long res = 0;
        a %= mod;
        while (b > 0) {
            if ((b & 1) == 1) res = (res + a) % mod;
            a = (a << 1) % mod;
            b >>= 1;
        }
        return res;
    }
}
```

**复杂度**：O(k·log max(mᵢ))，k 次合并操作。

**测试用例**：

   验证: 10%4=2, 10%6=4, 10%10=0 ✓

2) a=[2,4,5], m=[4,6,10]  →  无解
   原因: 合并第3式时 diff%g ≠ 0

3) a=[4,10], m=[6,14]
   推导: gcd(6,14)=2, diff=6, 6%2=0 ✓
   p: 6·?+14·?=2 → 6·5+14·(-2)=2 → p=5
   t = 6/2 × 5 mod 7 = 15 mod 7 = 1

### 4c. RSA-CRT 解密加速

#### 背景

RSA 解密计算 `m = c^d mod n`，其中 n = p·q（p、q 是大素数）。直接计算时指数 d 很长（≈ n 的位数），计算量很大。

**CRT 加速思路**：
1. 分别在模 p 和模 q 下计算：mₚ = c^d mod p, m_q = c^d mod q
2. 用 CRT 组合得到最终结果：m ≡ CRT(mₚ, m_q) mod p·q

**加速原理**：
- 直接解密：指数 d ≈ |n|，复杂度 O(log³n)
- CRT 加速：两个指数 d mod (p-1) 和 d mod (q-1)，每个约 |n|/2 位
- 总复杂度 ≈ 2 × O(log³(n/2)) ≈ O(log³n)/4，加速约 4 倍

**还需要考虑**：
- 在实际实现中，指数应简化：dₚ = d mod (p-1), d_q = d mod (q-1)
- 使用 Garner 算法组合结果更高效

**Java 代码**（完整 RSA-CRT 解密）：

```java
import java.math.BigInteger;
import java.security.SecureRandom;

public class RSACRTDecrypt {
    public static void main(String[] args) {
        // 生成密钥对（演示用，实际密钥小）
        SecureRandom rand = new SecureRandom();
        BigInteger p = BigInteger.probablePrime(512, rand);
        BigInteger q = BigInteger.probablePrime(512, rand);
        BigInteger n = p.multiply(q);
        BigInteger phi = p.subtract(BigInteger.ONE).multiply(q.subtract(BigInteger.ONE));
        BigInteger e = BigInteger.valueOf(65537);
        BigInteger d = e.modInverse(phi);

        System.out.println("=== RSA-CRT 解密加速 ===");
        System.out.println("p (bits): " + p.bitLength());
        System.out.println("q (bits): " + q.bitLength());
        System.out.println("n (bits): " + n.bitLength());

        // 待加密消息
        BigInteger msg = new BigInteger("20240615123456789");
        System.out.println("原始消息: " + msg);

        // 加密
        BigInteger cipher = msg.modPow(e, n);
        System.out.println("密文: " + cipher);

        // === 解密对比 ===

        // 1. 直接解密
        long t0 = System.nanoTime();
        BigInteger mDirect = cipher.modPow(d, n);
        long tDirect = System.nanoTime() - t0;

        // 2. CRT 加速解密
        t0 = System.nanoTime();
        BigInteger mCRT = rsaCRTDecrypt(cipher, p, q, d, e);
        long tCRT = System.nanoTime() - t0;

        System.out.println("\n直接解密结果: " + mDirect);
        System.out.println("CRT解密结果:  " + mCRT);
        System.out.println("结果一致: " + mDirect.equals(mCRT));
        System.out.println("直接解密耗时: " + tDirect / 1e6 + " ms");
        System.out.println("CRT 解密耗时: " + tCRT / 1e6 + " ms");
        System.out.println("加速比: " + String.format("%.2f", (double) tDirect / tCRT) + "x");
    }

    /**
     * RSA-CRT 解密
     *
     * 原理：
     * 1. 分别解密到模 p 和模 q
     * 2. 用 CRT 组合结果
     *
     * 关键的指数约简：
     * dₚ = d mod (p-1)  （费马小定理）
     * d_q = d mod (q-1)
     *
     * 组合使用 Garner 算法（比标准 CRT 更高效）
     */
    static BigInteger rsaCRTDecrypt(BigInteger c, BigInteger p, BigInteger q,
                                     BigInteger d, BigInteger e) {
        // 指数约简（费马小定理）
        BigInteger dp = d.mod(p.subtract(BigInteger.ONE));
        BigInteger dq = d.mod(q.subtract(BigInteger.ONE));

        // 分别在模 p、模 q 下解密
        BigInteger m1 = c.modPow(dp, p);
        BigInteger m2 = c.modPow(dq, q);

        // Garner 算法组合
        // x = m1 + p × ((m2 - m1) × p⁻¹ mod q)
        BigInteger invP = p.modInverse(q);  // p 模 q 的逆元
        BigInteger h = invP.multiply(m2.subtract(m1)).mod(q);
        if (h.signum() < 0) h = h.add(q);

        return m1.add(p.multiply(h));
    }
}
```

**复杂度**：
- 直接解密：O(log³n) ≈ O(3072³) 位操作
- CRT 加速：2 × O(log³(n/2)) ≈ O(1536³) × 2，约快 4 倍

**防御侧信道攻击**：
RSA-CRT 存在**Bellcore 攻击**：若模 p 下解密出错（如使用错误 dp），结果只在模 q 下正确，攻击者可分解 n。
防护方案：
```java
// 验证（防御错误注入攻击）
BigInteger result = rsaCRTDecrypt(c, p, q, d, e);
// 重加密验证
if (!result.modPow(e, n).equals(c)) {
    throw new SecurityException("CRT 解密校验失败，可能受到错误注入攻击");
}
```

### 4d. 大整数模运算 —— 分解为小模数并行处理

#### 思想

**Residue Number System (RNS)**：将大整数表示为一组余数，用 CRT 重建。

对于一个很大的整数 A 和很大模数 M（可能远超 64 位），可将 M 分解为小素数幂的乘积 M = m₁ × m₂ × ... × mₖ，然后在每个小模数下独立计算，最后用 CRT 合并。

#### 应用场景

1. **大数模乘**：A × B mod M，A、B、M 均为大数
2. **大数模幂**：A^B mod M
3. **多精度算术硬件加速**：每个 RNS 通道可并行处理

#### 示例：计算 123456789²³⁴⁵⁶⁷ mod 999999937

选用小模数集 {3, 5, 7, 11, 13, 17, 19, 23}（两两互质），
M = 3×5×7×11×13×17×19×23 = 111546435 > 999999937。

不直接做 mod 999999937 的大数运算，而是在每个小模数下做模幂，最后 CRT 合并。

**Java 代码**：

```java
import java.math.BigInteger;

public class LargeModuloWithCRT {
    public static void main(String[] args) {
        // 选择小模数集（两两互质，乘积大于目标模数）
        long[] smallModuli = {3, 5, 7, 11, 13, 17, 19, 23};
        BigInteger targetMod = new BigInteger("999999937");
        BigInteger base = new BigInteger("123456789");
        BigInteger exp = new BigInteger("23456789");

        // 验证选择：乘积 > targetMod
        BigInteger prod = BigInteger.ONE;
        for (long m : smallModuli) prod = prod.multiply(BigInteger.valueOf(m));
        System.out.println("小模数乘积: " + prod);
        System.out.println("乘积 > 目标模数: " + prod.compareTo(targetMod) > 0);

        // === 方法1：直接计算 ===
        long t0 = System.nanoTime();
        BigInteger directResult = base.modPow(exp, targetMod);
        long tDirect = System.nanoTime() - t0;
        System.out.println("\n直接计算结果: " + directResult);
        System.out.println("耗时: " + tDirect / 1e6 + " ms");

        // === 方法2：RNS-CRT 计算 ===
        t0 = System.nanoTime();
        BigInteger rnsResult = rnsModPow(base, exp, smallModuli, targetMod);
        long tRNS = System.nanoTime() - t0;
        System.out.println("RNS-CRT 结果: " + rnsResult);
        System.out.println("耗时: " + tRNS / 1e6 + " ms");
        System.out.println("一致: " + directResult.equals(rnsResult));
    }

    /**
     * 使用 RNS（Residue Number System）+ CRT 计算大数模幂
     *
     * 步骤：
     * 1. 对每个小模数 mᵢ，计算 base mod mᵢ、exp mod φ(mᵢ)（若 mᵢ 为素数）
     * 2. 在每个小模数下独立计算 base^exp mod mᵢ
     * 3. 使用 CRT 将结果合并，再 mod targetMod
     */
    static BigInteger rnsModPow(BigInteger base, BigInteger exp,
                                long[] moduli, BigInteger targetMod) {
        int k = moduli.length;
        long[] remainders = new long[k];

        // Step 1 & 2: 并行计算每个小模数下的余数
        // （实际可多线程并行）
        for (int i = 0; i < k; i++) {
            BigInteger mi = BigInteger.valueOf(moduli[i]);
            // 模数 mᵢ 下计算 base mod mᵢ
            BigInteger baseReduced = base.mod(mi);
            // 若 mᵢ 是素数，指数可约减 mod (mᵢ-1)（费马小定理）
            // 但为通用性，不做指数约减
            BigInteger resultInMi = baseReduced.modPow(exp, mi);
            remainders[i] = resultInMi.longValue();
        }

        // Step 3: CRT 合并
        BigInteger M = BigInteger.ONE;
        for (long m : moduli) M = M.multiply(BigInteger.valueOf(m));

        BigInteger result = BigInteger.ZERO;
        for (int i = 0; i < k; i++) {
            BigInteger mi = BigInteger.valueOf(moduli[i]);
            BigInteger Mi = M.divide(mi);
            BigInteger inv = Mi.mod(mi).modInverse(mi);
            BigInteger term = BigInteger.valueOf(remainders[i])
                .multiply(Mi).multiply(inv);
            result = result.add(term).mod(M);
        }

        // 最终对目标模数取模
        return result.mod(targetMod);
    }
}
```

**复杂度**：
- 直接模幂：O(log exp × log² targetMod)
- RNS-CRT：O(k·log exp × log² mᵢ)，每个小模数运算更快，且可**并行**
- 并行加速比 ≈ 1/k（理想情况），但实际受 k 限制

**实际收益**：当目标模数非常大（1024 位以上）时，分解为多个 64 位或 32 位小模数，可利用硬件原生整数运算，避免 BigInteger 开销。

### 4e. 日历对齐问题 —— 多周期求共同对齐时间

**问题描述**：
- 工作 A：每 a 天执行一次，第一次在 d₁ 天后
- 工作 B：每 b 天执行一次，第一次在 d₂ 天后
- 工作 C：每 c 天执行一次，第一次在 d₃ 天后
- 问：多少天后三个工作第一次**同时**执行？

**数学模型**：
```
x ≡ d₁ (mod a)
x ≡ d₂ (mod b)
x ≡ d₃ (mod c)
```

——这正是 CRT！

#### 示例：三个行星的周期对齐

```
行星 A：周期 3 年，上次对齐在 2019 年（该事件后 2 年）
行星 B：周期 5 年，上次对齐在 2019 年（该事件后 3 年）
行星 C：周期 7 年，上次对齐在 2019 年（该事件后 2 年）

问：什么时候三个行星再次对齐？
```

这就是"物不知数"的翻版：x ≡ 2 (mod 3), x ≡ 3 (mod 5), x ≡ 2 (mod 7)，x = 23 年后。

**一般化：多周期调度对齐**

```java
public class CalendarAlignment {
    public static void main(String[] args) {

                    + " = " + (result % periods[i]) + " (应=" + (offsets[i] % periods[i]) + ")");
            }
        } else {
            System.out.println("这些任务永远不会同时执行");
        }

        // 例2：三个行星周期（用 ExCRT，因为周期可能非互质）
        long[] planetPeriods = {8, 12, 18};   // 轨道周期（年）
        long[] planetStart  = {2, 4, 6};      // 上次对齐后的年数
        Long align = calendarAlign(planetPeriods, planetStart);
        if (align != null) {
            System.out.println("\n行星对齐时间: " + align + " 年后");
        }
    }

    /**
     * 日历对齐：求 x 使得 x ≡ offsets[i] (mod periods[i])
     */
    static Long calendarAlign(long[] periods, long[] offsets) {
        // 直接使用 ExCRT 处理（兼容非互质周期）
        return excrt(offsets, periods);
    }

    // ========== ExCRT 实现（同上节 4b）= ==========

    static class Congruence {
        long r, m;
        Congruence(long r, long m) {
            this.r = (r % m + m) % m;
            this.m = m;
        }
    }

    static Long excrt(long[] r, long[] m) {
        Congruence cur = new Congruence(r[0], m[0]);
        for (int i = 1; i < m.length; i++) {
            cur = merge(cur, new Congruence(r[i], m[i]));
            if (cur == null) return null;
        }
        return cur.r;
    }

    static Congruence merge(Congruence c1, Congruence c2) {
        long r1 = c1.r, m1 = c1.m;
        long r2 = c2.r, m2 = c2.m;
        long[] ext = exgcd(m1, m2);
        long g = ext[0], p = ext[1];
        long diff = r2 - r1;
        if ((diff % g + g) % g != 0) return null;
        long m2g = m2 / g;
        long t = ((diff / g) % m2g) * (p % m2g) % m2g;
        if (t < 0) t += m2g;
        long newM = m1 / g * m2;
        long newR = (r1 + m1 * t) % newM;
        if (newR < 0) newR += newM;
        return new Congruence(newR, newM);
    }

    static long[] exgcd(long a, long b) {
        if (b == 0) return new long[]{a, 1, 0};
        long[] res = exgcd(b, a % b);
        return new long[]{res[0], res[2], res[1] - (a / b) * res[2]};
    }
}
```

### 4f. 秘密共享 —— Shamir 门限方案

#### 核心思想

**Shamir 秘密共享**（Adi Shamir, 1979）将秘密 s 分割为 n 份，任意 t 份可恢复 s（(t, n)-门限方案）。

**与 CRT 的联系**：基于 CRT 的秘密共享方案（**Mignotte 方案**）使用两两互质的模数序列。

#### Mignotte 门限方案

设秘密 s，选择互质的模数序列 m₁ < m₂ < ... < mₙ。

**条件**：
- 任意 t 个模数乘积 > s
- 任意 t-1 个模数乘积 < s

**分片**：sᵢ = s mod mᵢ，i = 1,...,n

**恢复**：任意 t 个分片用 CRT 重建 s（在 t 个模数乘积大于 s 的保证下，重建结果就是 s 本身）

**少于 t 个分片**：乘积小于 s，无法恢复（信息论安全）。

#### 示例实现

```java
import java.math.BigInteger;
import java.util.*;

public class MignotteSecretSharing {
    private BigInteger[] moduli;   // 互质模数序列
    private int n;                 // 分片数
    private int t;                 // 门限值
    private BigInteger productT;   // 最大 t 个模数乘积
    private BigInteger productT1;  // 最大 t-1 个模数乘积

    /**
     * 初始化 Mignotte 方案
     * @param n 总分片数
     * @param t 恢复需要的门限
     * @param secretBitLength 秘密的比特长度
     */
    public MignotteSecretSharing(int n, int t, int secretBitLength) {
        this.n = n;
        this.t = t;
        generateModuli(secretBitLength);
    }

    /**
     * 生成互质模数序列
     * 要求：product_{t} > s > product_{t-1}
     * 其中 product_{t} 是最大的 t 个模数乘积
     */
    private void generateModuli(int bitLen) {
        // 简化实现：使用连续素数
        // 实际方案需要更精心的选择
        moduli = new BigInteger[n];
        int idx = 0;
        BigInteger prime = BigInteger.valueOf(2);
        while (idx < n) {
            if (prime.isProbablePrime(100)) {
                moduli[idx] = prime;
                idx++;
            }
            prime = prime.nextProbablePrime();
        }

        // 计算门限乘积
        BigInteger[] sorted = moduli.clone();
        Arrays.sort(sorted);

        productT = BigInteger.ONE;
        for (int i = n - t; i < n; i++) {
            productT = productT.multiply(sorted[i]);
        }

        productT1 = BigInteger.ONE;
        for (int i = 0; i < t - 1; i++) {
            productT1 = productT1.multiply(sorted[i]);
        }
    }

    /**
     * 分割秘密
     * @param secret 秘密值（须满足 productT1 < secret < productT）
     */
    public BigInteger[] split(BigInteger secret) {
        if (secret.compareTo(productT1) <= 0 || secret.compareTo(productT) >= 0) {
            throw new IllegalArgumentException(
                "秘密必须在 " + productT1 + " 和 " + productT + " 之间");
        }

        BigInteger[] shares = new BigInteger[n];
        for (int i = 0; i < n; i++) {
            shares[i] = secret.mod(moduli[i]);
        }
        return shares;
    }

    /**
     * 恢复秘密（使用 CRT）
     * @param shares 分片子集 (id, value)
     * @return 恢复的秘密
     */
    public BigInteger recover(Map<Integer, BigInteger> shares) {
        if (shares.size() < t) {
            throw new IllegalArgumentException("至少需要 " + t + " 个分片");
        }

        int k = shares.size();
        BigInteger[] r = new BigInteger[k];
        BigInteger[] m = new BigInteger[k];

        int idx = 0;
        for (Map.Entry<Integer, BigInteger> entry : shares.entrySet()) {
            r[idx] = entry.getValue();
            m[idx] = moduli[entry.getKey()];
            idx++;
        }

        // CRT 重建
        BigInteger M = BigInteger.ONE;
        for (BigInteger mi : m) M = M.multiply(mi);

        BigInteger result = BigInteger.ZERO;
        for (int i = 0; i < k; i++) {
            BigInteger Mi = M.divide(m[i]);
            BigInteger inv = Mi.mod(m[i]).modInverse(m[i]);
            result = result.add(r[i].multiply(Mi).multiply(inv)).mod(M);
        }

        return result;
    }

    public BigInteger getProductT() { return productT; }
    public BigInteger getProductT1() { return productT1; }
    public BigInteger[] getModuli() { return moduli; }

    // ========== 演示 ==========

    public static void main(String[] args) {
        // (3,5)-门限：5 个分片，任意 3 个可恢复
        MignotteSecretSharing scheme = new MignotteSecretSharing(5, 3, 32);
        BigInteger secret = new BigInteger("123456789");
        System.out.println("原始秘密: " + secret);
        System.out.println("模数: " + Arrays.toString(scheme.getModuli()));
        System.out.println("product_T:  " + scheme.getProductT());
        System.out.println("product_T1: " + scheme.getProductT1());

        // 分割
        BigInteger[] shares = scheme.split(secret);
        System.out.println("\n5 个分片:");
        for (int i = 0; i < shares.length; i++) {
            System.out.println("  分片[" + i + "] = " + shares[i]);
        }

        // 用 3 个分片恢复
        Map<Integer, BigInteger> subset = new HashMap<>();
        subset.put(0, shares[0]);
        subset.put(2, shares[2]);
        subset.put(4, shares[4]);

        BigInteger recovered = scheme.recover(subset);
        System.out.println("\n用分片 {0,2,4} 恢复: " + recovered);
        System.out.println("恢复正确: " + secret.equals(recovered));

        // 只给 2 个分片（无法恢复）
        Map<Integer, BigInteger> subset2 = new HashMap<>();
        subset2.put(0, shares[0]);
        subset2.put(1, shares[1]);
        try {
            BigInteger wrong = scheme.recover(subset2);
            System.out.println("\n用分片 {0,1} 恢复: " + wrong);
            System.out.println("是否等于原始秘密: " + secret.equals(wrong));
        } catch (IllegalArgumentException e) {
            System.out.println("\n分片不足 " + t + " 个: " + e.getMessage());
        }
    }
}
```

### 4g. 高次同余方程 —— 模数分解为素数幂

#### 核心思想

解高次同余方程 `f(x) ≡ 0 (mod m)` 时，若 m 不是素数幂，可利用 CRT 分解：

1. 将 m 分解为素数幂：m = p₁^{e₁} × p₂^{e₂} × ... × pₖ^{eₖ}
2. 在每个素数幂模下求解：f(x) ≡ 0 (mod pᵢ^{eᵢ})
3. 用 CRT 合并各模数下的解

**原理**：
```
f(x) ≡ 0 (mod m)  ⇔ 对每个 i: f(x) ≡ 0 (mod pᵢ^{eᵢ})
```
因为若 m = Π pᵢ^{eᵢ} 且各因子互质，x 满足所有条件当且仅当满足 m。

#### 示例：x² ≡ 1 (mod 36)

```
36 = 2² × 3² = 4 × 9

模 4 下：x² ≡ 1 (mod 4) → x ≡ 1 or 3 (mod 4)
模 9 下：x² ≡ 1 (mod 9) → x ≡ 1 or 8 (mod 9)

CRT 合并 4 种组合：
  (1 mod 4, 1 mod 9) → x ≡ 1 (mod 36)
  (1 mod 4, 8 mod 9) → x ≡ 17 (mod 36)
  (3 mod 4, 1 mod 9) → x ≡ 19 (mod 36)
  (3 mod 4, 8 mod 9) → x ≡ 35 (mod 36)

原方程在模 36 下有 4 个解：1, 17, 19, 35
```

#### 通用实现

```java
import java.math.BigInteger;
import java.util.*;

public class HighOrderCongruenceCRT {
    public static void main(String[] args) {
        // 示例：x² ≡ 1 (mod 36)
        Polynomial poly = new Polynomial(new long[]{1, 0, -1}); // x² - 1 = 0
        long m = 36;
        List<Long> solutions = solvePolynomialMod(poly, m);
        System.out.println("x² ≡ 1 (mod " + m + ") 的解: " + solutions);
        for (long x : solutions) {
            System.out.println("  验证: " + x + "² mod " + m + " = " + (x * x % m));
        }

        System.out.println();

        // 示例：x² + 3x + 2 ≡ 0 (mod 30)
        Polynomial poly2 = new Polynomial(new long[]{2, 3, 1}); // x² + 3x + 2 = 0
        long m2 = 30;
        List<Long> solutions2 = solvePolynomialMod(poly2, m2);
        System.out.println("x² + 3x + 2 ≡ 0 (mod " + m2 + ") 的解: " + solutions2);
        for (long x : solutions2) {
            long val = (x * x + 3 * x + 2) % m2;
            System.out.println("  验证: x=" + x + ", val mod " + m2 + " = " + val);
        }
    }

    /**
     * 解多项式同余方程 f(x) ≡ 0 (mod m)
     * 步骤：
     * 1. 分解 m 为素数幂
     * 2. 在每个素数幂下求解
     * 3. CRT 合并所有组合
     */
    static List<Long> solvePolynomialMod(Polynomial poly, long m) {
        // Step 1: 因式分解 m = Π pᵢ^{eᵢ}
        Map<Long, Integer> factors = factorize(m);
        List<Map.Entry<Long, Integer>> factorList = new ArrayList<>(factors.entrySet());

        int k = factorList.size();
        List<List<Long>> solutionsPerMod = new ArrayList<>();

        // Step 2: 在每个素数幂下求解
        for (Map.Entry<Long, Integer> entry : factorList) {
            long prime = entry.getKey();
            int exp = entry.getValue();
            long mod = (long) Math.pow(prime, exp);
            solutionsPerMod.add(solvePolynomialModPrimePower(poly, prime, exp));
        }

        // Step 3: CRT 合并（笛卡尔积）
        List<Long> result = new ArrayList<>();
        cartesianCRT(poly, solutionsPerMod, factorList, 0,
                     new long[k], result, m);
        Collections.sort(result);
        return result;
    }

    /**
     * 解 f(x) ≡ 0 (mod p^e)
     * 使用 Hensel 引理提升：先解模 p，再逐次提升到模 p^e
     */
    static List<Long> solvePolynomialModPrimePower(Polynomial poly, long p, int e) {
        List<Long> solModP = solvePolynomialModPrime(poly, p);
        if (solModP.isEmpty()) return solModP;

        List<Long> result = new ArrayList<>();
        long mod = p;

        for (long x0 : solModP) {
            long x = x0;
            long curMod = p;

            // Hensel 提升
            for (int i = 1; i < e; i++) {
                curMod *= p;
                // f(x) + f'(x)·t ≡ 0 (mod curMod)
                long fVal = poly.evaluate(x, curMod);
                long fDeriv = poly.evaluateDerivative(x, p); // 只需模 p

                if (fDeriv % p != 0) {
                    // 非奇异提升：唯一解
                    long inv = modInverse(fDeriv % p, p);
                    long t = ((-fVal / mod) % p * inv) % p;
                    if (t < 0) t += p;
                    x = x + mod * t;
                } else if (fVal % curMod == 0) {
                    // 奇异但可解：多解
                    // 简化处理：直接枚举
                    //（完整实现略，比赛时模数较小可暴力枚举）
                }
            }
            result.add(x % curMod);
        }
        return result;
    }

    /**
     * 解 f(x) ≡ 0 (mod p)，p 为素数
     * 小素数用枚举，大素数用更高效算法（略）
     */
    static List<Long> solvePolynomialModPrime(Polynomial poly, long p) {
        List<Long> solutions = new ArrayList<>();
        for (long x = 0; x < p; x++) {
            if (poly.evaluate(x, p) == 0) {
                solutions.add(x);
            }
        }
        return solutions;
    }

    /**
     * CRT 笛卡尔积合并
     */
    static void cartesianCRT(Polynomial poly,
                              List<List<Long>> sols,
                              List<Map.Entry<Long, Integer>> factors,
                              int idx, long[] current,
                              List<Long> result, long fullMod) {
        if (idx == sols.size()) {
            // 用 CRT 合并 current 中的余数
            long[] remainders = new long[current.length];
            long[] moduli = new long[current.length];
            for (int i = 0; i < current.length; i++) {
                remainders[i] = current[i];
                moduli[i] = (long) Math.pow(factors.get(i).getKey(),
                                           factors.get(i).getValue());
            }
            long x = crt(remainders, moduli);
            if (x >= 0) {
                result.add(x % fullMod);
            }
            return;
        }

        for (long sol : sols.get(idx)) {
            current[idx] = sol;
            cartesianCRT(poly, sols, factors, idx + 1, current, result, fullMod);
        }
    }

    // ========== 工具函数 ==========

    /** 因式分解为素数幂 */
    static Map<Long, Integer> factorize(long n) {
        Map<Long, Integer> factors = new HashMap<>();
        for (long p = 2; p * p <= n; p++) {
            int cnt = 0;
            while (n % p == 0) {
                n /= p;
                cnt++;
            }
            if (cnt > 0) factors.put(p, cnt);
        }
        if (n > 1) factors.put(n, 1);
        return factors;
    }

    static long crt(long[] remainders, long[] moduli) {
        long M = 1;
        for (long m : moduli) M *= m;
        long result = 0;
        for (int i = 0; i < moduli.length; i++) {
            long Mi = M / moduli[i];
            long inv = modInverse(Mi % moduli[i], moduli[i]);
            if (inv < 0) return -1;
            result = (result + remainders[i] * (Mi % M) % M * inv) % M;
        }
        return (result + M) % M;
    }

    static long modInverse(long a, long m) {
        long[] res = exgcd(a, m);
        return res[0] == 1 ? (res[1] % m + m) % m : -1;
    }

    static long[] exgcd(long a, long b) {
        if (b == 0) return new long[]{a, 1, 0};
        long[] res = exgcd(b, a % b);
        return new long[]{res[0], res[2], res[1] - (a / b) * res[2]};
    }

    // ========== 多项式辅助类 ==========

    static class Polynomial {
        long[] coeffs; // 从低次到高次: coeffs[0] + coeffs[1]·x + ...

        Polynomial(long[] coeffs) {
            this.coeffs = coeffs.clone();
        }

        long evaluate(long x, long mod) {
            long result = 0;
            long power = 1;
            for (long c : coeffs) {
                result = (result + c * power) % mod;
                power = (power * x) % mod;
            }
            return result;
        }

        long evaluateDerivative(long x, long mod) {
            long result = 0;
            long power = 1;
            for (int i = 1; i < coeffs.length; i++) {
                result = (result + i * coeffs[i] % mod * power) % mod;
                power = (power * x) % mod;
            }
            return result;
        }
    }
}

  验证: 35² mod 36 = 1

x² + 3x + 2 ≡ 0 (mod 30) 的解: [1, 10, 13, 16, 19, 28]
因式分解 30 = 2 × 3 × 5
模 2: x² + 3x + 2 ≡ x² + x ≡ 0 → x ≡ 0,1 (mod 2)
模 3: x² + 3x + 2 ≡ x² + 2 ≡ 0 → x ≡ 1,2 (mod 3)
模 5: x² + 3x + 2 ≡ x² + 3x + 2 ≡ 0 → x ≡ 1,3 (mod 5)
CRT 合并 2×2×2 = 8 种组合，筛去重复得到 6 个解
```

### 4h. 线性同余方程组通解结构

#### 解空间描述

对于线性同余方程组：
```
x ≡ a₁ (mod m₁)
x ≡ a₂ (mod m₂)
...
x ≡ aₖ (mod mₖ)
```

**有解条件**：对所有 i ≠ j，有 gcd(mᵢ, mⱼ) | (aᵢ - aⱼ)

**通解结构**：
- 若存在特解 x₀，则通解为 **x ≡ x₀ (mod L)**，其中 L = lcm(m₁, m₂, ..., mₖ)
- 即在模 L 的完全剩余系中恰有 **1 个解**

#### 解的个数分析

| 条件 | 在模 L 下解的个数 | 说明 |
|------|-----------------|------|
| 全部互质 + 一致 | 1 | 标准 CRT 情形 |
| 非互质但有解 | 1 | ExCRT 情形 |
| 非互质且无解 | 0 | 方程组矛盾 |
| 互质但增广（多余方程） | 1 | 增广一致性自动保证 |

> **关键结论**：线性同余方程组在**相容**时，解空间是一个**模 L 的同余类**。

#### 多个解空间的代数结构

考虑更一般的形式：
- 方程组 (I)：x ≡ aᵢ (mod mᵢ)，i=1..k
- 方程组 (II)：x ≡ bⱼ (mod nⱼ)，j=1..l

**模运算意义**：
若记 S₁ 为 (I) 的解集 { x | x ≡ x₁ (mod L₁) }，S₂ 为 (II) 的解集 { x | x ≡ x₂ (mod L₂) }，
则 S₁ ∩ S₂ 的解需满足：
```
x ≡ x₁ (mod L₁)
x ≡ x₂ (mod L₂)
```
即再次使用 ExCRT 合并！

#### Java 实现：CRT 系统求解器

```java
import java.math.BigInteger;
import java.util.*;

/**
 * 线性同余方程组求解器
 * 支持任意多方程，处理非互质情况
 */
public class CRTSolver {
    private List<Congruence> equations;

    public CRTSolver() {
        equations = new ArrayList<>();
    }

    public void addEquation(long remainder, long mod) {
        equations.add(new Congruence(remainder, mod));
    }

    /**
     * 求解整个方程组
     * @return 解的描述：x ≡ result (mod modNum)，或 null 表示无解
     */
    public Congruence solve() {
        if (equations.isEmpty()) return null;

        Congruence cur = equations.get(0);
        for (int i = 1; i < equations.size(); i++) {
            cur = merge(cur, equations.get(i));
            if (cur == null) return null;
        }
        return cur;
    }

    /**
     * 描述解空间
     */
    public String describeSolution() {
        Congruence sol = solve();
        if (sol == null) return "方程组无解 ❌";

        StringBuilder sb = new StringBuilder();
        sb.append("解存在: x ≡ ").append(sol.r)
          .append(" (mod ").append(sol.m).append(")\n");
        sb.append("在模 ").append(sol.m).append(" 下有唯一解 ").append(sol.r).append("\n");
        sb.append("通解：x = ").append(sol.r)
          .append(" + ").append(sol.m).append("·t, t ∈ ℤ");
        return sb.toString();
    }

    // ========== 内部实现 ==========

    static class Congruence {
        long r, m;
        Congruence(long r, long m) {
            this.r = (r % m + m) % m;
            this.m = m;
        }
    }

    static Congruence merge(Congruence c1, Congruence c2) {
        long r1 = c1.r, m1 = c1.m;
        long r2 = c2.r, m2 = c2.m;
        long[] ext = exgcd(m1, m2);
        long g = ext[0], p = ext[1];
        long diff = r2 - r1;
        if ((diff % g + g) % g != 0) return null;
        long m2g = m2 / g;
        long t = ((diff / g) % m2g) * (p % m2g) % m2g;
        if (t < 0) t += m2g;
        long newM = m1 / g * m2;
        long newR = (r1 + m1 % newM * t % newM) % newM;
        if (newR < 0) newR += newM;
        return new Congruence(newR, newM);
    }

    static long[] exgcd(long a, long b) {
        if (b == 0) return new long[]{a, 1, 0};
        long[] res = exgcd(b, a % b);
        return new long[]{res[0], res[2], res[1] - (a / b) * res[2]};
    }

    public static void main(String[] args) {
        // 例1：标准 CRT
        CRTSolver solver1 = new CRTSolver();
        solver1.addEquation(2, 3);
        solver1.addEquation(3, 5);
        solver1.addEquation(2, 7);
        System.out.println("例1 - 物不知数:");
        System.out.println(solver1.describeSolution());
        System.out.println();

        // 例2：ExCRT 非互质
        CRTSolver solver2 = new CRTSolver();
        solver2.addEquation(2, 4);
        solver2.addEquation(4, 6);
        System.out.println("例2 - 非互质(有解):");
        System.out.println(solver2.describeSolution());
        System.out.println();

        // 例3：无解
        CRTSolver solver3 = new CRTSolver();
        solver3.addEquation(2, 4);
        solver3.addEquation(3, 6);
        System.out.println("例3 - 无解:");
        System.out.println(solver3.describeSolution());
        System.out.println();

        // 例4：冗余方程（增广）
        CRTSolver solver4 = new CRTSolver();
        solver4.addEquation(2, 3);
        solver4.addEquation(3, 5);
        solver4.addEquation(2, 7);
        solver4.addEquation(2, 3); // 冗余
        solver4.addEquation(23, 105); // 冗余且等价
        System.out.println("例4 - 冗余方程:");
        System.out.println(solver4.describeSolution());
    }
}
```
