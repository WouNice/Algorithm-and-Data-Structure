# 最大公约数与扩展欧几里得算法

> **核心脉络**：欧几里得算法（公元前 300 年）→ 扩展欧几里得（求解 `ax+by=gcd(a,b)`）→ 丢番图方程通解 → 模线性方程 & 逆元 & 同余方程组 → 密码学（RSA）

## 诞生背景与核心原理

### 欧几里得算法（Euclidean Algorithm）的历史

欧几里得算法最早出现在欧几里得的《几何原本》（公元前 300 年）第七卷中，是数学史上最古老且仍在广泛使用的算法之一。它的英文名称 "Euclidean" 即来源于欧几里得（Euclid，希腊文 Ευκλείδης）。

**历史里程碑**：

| 时间 | 人物/著作 | 贡献 |
|------|-----------|------|
| 约公元前 300 年 | 欧几里得《几何原本》 | 提出辗转相除法的几何版本（线段长度公度量） |
| 约公元 500 年 | 印度数学家 Aryabhata | 称为"粉碎机"（Kuṭṭaka），用于求解丢番图方程 |
| 1621 年 | Claude Gaspard Bachet de Méziriac | 重新发现并发表丢番图的《算术》，为扩展欧几里得奠定基础 |
| 1740 年 | 欧拉（Euler） | 将其用于连分数理论，系统分析 |
| 1848 年 | 斯坦纳（J. Stein） | 提出二进制 GCD 算法（Stein's algorithm / binary GCD） |
| 1969 年 | Knuth《计算机程序设计艺术》 | 系统分析算法复杂度，推广到计算机科学 |
| 1977 年 | Rivest-Shamir-Adleman | RSA 密码体系用扩展欧几里得求私钥 d |

### 辗转相除法——完备数学证明

**定理**：设 `a, b` 为不全为零的整数，则：

```
gcd(a, b) = gcd(b, a mod b)
```

**证明（分两步）**：

**1. 证明 `gcd(a, b)` 和 `gcd(b, r)` 的所有公因子相同**

由带余除法：`a = b × q + r`，其中 `q = ⌊a/b⌋`，`0 ≤ r < b`。

设 `d` 是 `a` 与 `b` 的任意公因子，即 `d | a` 且 `d | b`。

由于 `r = a - bq`，且 `d | a`，`d | b`，因此：
```
d | (a - bq) = r
```
所以 `d` 也是 `b` 与 `r` 的公因子。

反之，设 `d` 是 `b` 与 `r` 的公因子，即 `d | b` 且 `d | r`。

由于 `a = bq + r`，且 `d | b`，`d | r`，因此：
```
d | (bq + r) = a
```
所以 `d` 也是 `a` 与 `b` 的公因子。

**2. 结论**

`a, b` 的所有公因子集合 = `b, r` 的所有公因子集合
⇒ 最大公因子也相同，即 `gcd(a, b) = gcd(b, r) = gcd(b, a mod b)`。

**算法终止性证明**：

每次递归中，`|b|` 严格递减（因为 `0 ≤ r < |b|`），而参数为非负整数序列，故必在有限步内达到 `b = 0`。这是**严格递减论证**的经典案例。

**时间复杂度**：

- **最坏情况**：输入为**连续斐波那契数对**时，递归深度最大。`gcd(Fₙ, Fₙ₊₁)` 需要大约 `n` 次取模。
- **复杂度**：`O(log min(|a|, |b|))` 次取模运算，具体约为 `4.8 × log₁₀(min(a,b)) + 0.67` 次除法（拉梅定理，1844 年）。
- **位运算复杂度**（考虑大整数乘法）：`O(n²)`，其中 `n` 是数字的二进制位数。更精确的亚二次算法（如 Schönhage 法）最优可达 `O(n log² n log log n)`。

> **拉梅（Gabriel Lamé）定理**：用欧几里得算法计算 `gcd(a, b)`（`a ≥ b`）所需除法次数不超过 `5 × log₁₀(b) + 1`。证明基于：最坏情况是斐波那契数 `Fₖ₊₂`。

**可视化流程**：

```
gcd(123456789, 987654321)

987654321 = 8 × 123456789 + 9        → gcd(123456789, 9)
123456789 = 13717421 × 9 + 0         → gcd(9, 0)
→ g = 9
```

### 扩展欧几里得算法——递推推导与回溯公式证明

**问题**：求 `(x, y)` 使得：
```
ax + by = gcd(a, b)
```

**递归推导**：

定义函数 `exgcd(a, b) → (g, x, y)`。

**基准情况**（`b = 0`）：
```
a × 1 + 0 × 0 = a = gcd(a, 0)
⇒ g = a, x = 1, y = 0
```

**归纳步骤**（`b ≠ 0`）：

假设我们已经递归求解了 `exgcd(b, a mod b) → (g, x₁, y₁)`，满足：
```
b × x₁ + (a mod b) × y₁ = g
```

现在我们要解：
```
a × x + b × y = g
```

关键变换：将 `a mod b` 展开为 `a - ⌊a/b⌋ × b`。

代入递归结果：
```
b × x₁ + (a - ⌊a/b⌋ × b) × y₁ = g
```

展开：
```
b × x₁ + a × y₁ - ⌊a/b⌋ × b × y₁ = g
```

合并含 `a` 和含 `b` 的项：
```
a × y₁ + b × (x₁ - ⌊a/b⌋ × y₁) = g
```

对比目标方程 `a × x + b × y = g`，得回溯公式：

```
x = y₁
y = x₁ - ⌊a/b⌋ × y₁
```

**回溯过程图解**（以 gcd(47, 30) = 1 为例）：

### 线性丢番图方程 `ax + by = c` 的通解理论

#### 解的存在条件

```
方程 ax + by = c 有整数解 ⇔ gcd(a, b) | c
```

**证明**：
- **必要性**：设 `g = gcd(a, b)`，则 `g | a` 且 `g | b`，所以 `g | (ax + by) = c`。
- **充分性**：先用 exgcd 找到 `(x₀, y₀)` 使 `ax₀ + by₀ = g`，则 `x' = x₀ × (c/g)`，`y' = y₀ × (c/g)` 即为特解。

#### 通解公式

如果 `(x', y')` 是一组特解，则所有解为：

```
x = x' + (b/g) × t
y = y' - (a/g) × t    (t ∈ ℤ)
```

**证明**：设 `(X, Y)` 是另一组解，则有：
```
aX + bY = c
ax' + by' = c
```
相减得 `a(X - x') + b(Y - y') = 0`，即 `a(X - x') = -b(Y - y')`。

两边除以 `g = gcd(a, b)`：
```
(a/g)(X - x') = -(b/g)(Y - y')
```

由于 `gcd(a/g, b/g) = 1`，所以 `b/g | (X - x')`，令 `X - x' = (b/g) × t`，代入得 `(a/g) × (b/g) × t = -(b/g)(Y - y')`，即 `Y - y' = -(a/g) × t`。

### 欧拉函数与 GCD 的深层联系

**性质**：对于正整数 `n`，有：
```
Σ_{i=1}^{n} gcd(i, n) = Σ_{d|n} d × φ(n/d)
```
其中 `φ` 是欧拉函数。这个公式在后面"典型题目"中会用到。

## 核心问题与适用边界

### 正负数与零的处理

**零的 GCD**：数学定义中，`gcd(a, 0) = |a|`。在算法实现中：
- 当 `b = 0` 时返回 `a`
- `gcd(0, 0)` 有争议——数学上未定义，通常返回 0（Java 的 `BigInteger` 返回 0）

**负数的 GCD**：
- 数学定义：`gcd(a, b) = gcd(|a|, |b|)`
- 算法实现：取绝对值后计算，或利用 `gcd(a, b) = gcd(|a|, |b|)`

```java
// 处理负数的最佳实践
public static long gcd(long a, long b) {
    if (a == 0) return Math.abs(b);
    if (b == 0) return Math.abs(a);
    a = Math.abs(a);
    b = Math.abs(b);
    while (b != 0) {
        long temp = a % b;
        a = b;
        b = temp;
    }
    return a;
}
```

### 大数（BigInteger）处理

对于超过 64 位（`-2^63 ~ 2^63-1`）的整数，必须使用 `BigInteger`：

```java
import java.math.BigInteger;

BigInteger gcd = BigInteger.valueOf(a).gcd(BigInteger.valueOf(b));

// 扩展欧几里得
BigInteger[] exgcd(BigInteger a, BigInteger b) {
    if (b.equals(BigInteger.ZERO))
        return new BigInteger[]{a, BigInteger.ONE, BigInteger.ZERO};
    BigInteger[] res = exgcd(b, a.mod(b));
    BigInteger x1 = res[1], y1 = res[2];
    return new BigInteger[]{
        res[0],
        y1,
        x1.subtract(a.divide(b).multiply(y1))
    };
}
```

Java 8+ 原生支持 `BigInteger.gcd()`（内部使用二进制 GCD 优化）。

### 递归 vs 迭代——栈溢出问题

**递归深度的极限**：

| 内存设置 | 大约最大递归深度 | 约可处理的数字位数 |
|----------|-----------------|-------------------|
| 默认（-Xss512k~1M） | 5,000~10,000 | 约 10^1,000 |
| 默认（Windows x64） | ~8,000 | 约 10^1,600 |
| -Xss2M | ~20,000 | 约 10^4,000 |
| -Xss10M | ~100,000 | 约 10^20,000 |

**拉梅定理的定量分析**：

以 64 位 `long` 为例（最大约 `9.22 × 10¹⁸`），欧几里得算法的最坏递归深度约为：
```
5 × log₁₀(9.22 × 10¹⁸) + 1 ≈ 5 × 19 + 1 = 96 次
```
所以对于 64 位整数，递归深度不超过 100，栈溢出风险**极低**。

但对于 `BigInteger`（千位、万位数），深度可轻松达到数千上万，此时**必须使用迭代版**。

**结论**：64 位以内用递归简洁优雅，`BigInteger` 必须迭代。

## 高效实现与关键优化

### GCD 的三种实现

```java
// -------- 方法 1：递归（简洁）--------
public static long gcdRec(long a, long b) {
    if (a < 0) a = -a;
    if (b < 0) b = -b;
    return b == 0 ? a : gcdRec(b, a % b);
}

// -------- 方法 2：迭代（推荐，避免栈溢出）--------
public static long gcdIter(long a, long b) {
    if (a < 0) a = -a;
    if (b < 0) b = -b;
    while (b != 0) {
        long temp = a % b;
        a = b;
        b = temp;
    }
    return a;
}

// -------- 方法 3：位运算 gcd（非负）--------
public static long gcdBit(long a, long b) {
    if (a == 0) return b;
    if (b == 0) return a;
    // 提取 2 的幂次公因子
    int shift = Long.numberOfTrailingZeros(a | b);
    a >>= Long.numberOfTrailingZeros(a);
    do {
        b >>= Long.numberOfTrailingZeros(b);
        if (a > b) { long t = a; a = b; b = t; }
        b -= a;
    } while (b != 0);
    return a << shift;
}
```

**性能对比**（以 10⁶ 轮循环测试）：

| 算法 | 时间（相对） | 适用场景 |
|------|-------------|---------|
| 递归取模 | 1.0×（基准） | 64 位以内小数字 |
| 迭代取模 | 0.95× | 通用 |
| 二进制 GCD（Stein） | 1.2×~2× | 大整数（10³+位） |
| Java BigInteger.gcd() | 内部用二进制 GCD | 超大数据 |

> 注：对于 64 位以内整数，取模版本的简单性往往胜过二进制 GCD 的微优化；对于 `BigInteger`（千位以上），二进制 GCD 因为避免了大数除法而显著更快。

### Stein 算法（二进制 GCD）——深度解析

**核心思想**：用位运算替换取模运算。对于大数，位运算比取模快得多。

**核心规则**（设 `a, b` 均为非负）：

```
gcd(a, 0) = a
gcd(0, b) = b
gcd(a, b) = 2 × gcd(a/2, b/2)   // a, b 均为偶数
gcd(a, b) = gcd(a/2, b)          // a 偶 b 奇
gcd(a, b) = gcd(a, b/2)          // a 奇 b 偶
gcd(a, b) = gcd(|a-b|, min(a,b)) // a, b 均为奇数
```

**证明**：因为奇数 × 偶数不含因子 2，而偶数含因子 2，可以安全移除。

**完整 Java 实现（非负整数、大数友好）**：

```java
/**
 * Stein's Binary GCD Algorithm
 * 适用范围：long 或 BigInteger，当取模昂贵时特别有效
 */
public static long binaryGcd(long a, long b) {
    if (a == 0) return b;
    if (b == 0) return a;

    // 1. 提取 2 的公共幂次因子
    int shift = Long.numberOfTrailingZeros(a | b);
    // 在 Java 中 numberOfTrailingZeros(0) = 64，所以需预先处理零

    // 2. 将 a 的因子 2 全部移除
    a >>= Long.numberOfTrailingZeros(a);

    while (b != 0) {
        // 3. 将 b 的因子 2 全部移除
        b >>= Long.numberOfTrailingZeros(b);

        if (a > b) {
            long t = a;
            a = b;
            b = t;
        }
        // 4. 两奇数相减得偶数（关键优化）
        b -= a;
    }

    // 5. 恢复公共因子
    return a << shift;
}
```

**优点**：
- 用 `>>`、`&` 代替 `%`，对大整数（千位以上）快 2~5 倍
- 没有除法，适合硬件实现

**缺点**：
- 代码比辗转相除法复杂
- 对小整数（64 位内）不一定更快（CPU 的取模指令已高度优化）

### 扩展欧几里得的三种实现

```java
// -------- 方法 1：递归（教科书）--------
public static long[] exgcdRec(long a, long b) {
    if (b == 0) return new long[]{a, 1, 0};  // [g, x, y]
    long[] res = exgcdRec(b, a % b);
    long x1 = res[1], y1 = res[2];
    return new long[]{res[0], y1, x1 - a / b * y1};
}

// -------- 方法 2：迭代（推荐，避免栈溢出）--------
public static long[] exgcdIter(long a, long b) {
    long x0 = 1, y0 = 0, x1 = 0, y1 = 1;
    long a0 = a, b0 = b;
    while (b0 != 0) {
        long q = a0 / b0;
        long temp = a0 % b0;
        a0 = b0;
        b0 = temp;

        temp = x0 - q * x1;
        x0 = x1;
        x1 = temp;

        temp = y0 - q * y1;
        y0 = y1;
        y1 = temp;
    }
    return new long[]{a0, x0, y0};
}

// -------- 方法 3：带符号处理的完整版 ---------
public static class ExgcdResult {
    public long g, x, y;
    public ExgcdResult(long g, long x, long y) {
        this.g = g; this.x = x; this.y = y;
    }
}

/**
 * 通用扩展欧几里得（自动处理负数）
 * 保证 ax + by = g 且 |x| ≤ |b/g|, |y| ≤ |a/g|
 */
public static ExgcdResult exgcdSigned(long a, long b) {
    boolean aNeg = a < 0;
    boolean bNeg = b < 0;
    long[] res = exgcdIter(Math.abs(a), Math.abs(b));
    if (aNeg) res[1] = -res[1];
    if (bNeg) res[2] = -res[2];
    return new ExgcdResult(res[0], res[1], res[2]);
}
```

### 线性丢番图方程通解——完整推导 + 最小非负解

**方程**：`ax + by = c`

**步骤**：

1. 检查可解性：`g = gcd(|a|, |b|)`，若 `c % g != 0`，无解
2. 用 exgcd 求 `(x₀, y₀)` 满足 `ax₀ + by₀ = g`
3. 特解：`x' = x₀ × (c/g)`，`y' = y₀ × (c/g)`
4. 通解：`x = x' + (b/g) × t`，`y = y' - (a/g) × t`，`t ∈ ℤ`

**最小非负整数解（x 的最小非负值）**：

```java
/**
 * 求 ax + by = c 的通解形式，并取 x 的最小非负整数解
 * 返回 [x_minNonNeg, (b/g), (a/g)] 即 [特解的x, dx步长, dy步长]
 * 若无解返回 null
 */
public static long[] solveDiophantine(long a, long b, long c) {
    long absA = Math.abs(a);
    long absB = Math.abs(b);
    long[] ext = exgcdIter(absA, absB);
    long g = ext[0];             // gcd
    ext[1] = a < 0 ? -ext[1] : ext[1];
    ext[2] = b < 0 ? -ext[2] : ext[2];

    if (c % g != 0) return null;

    long x0 = ext[1] * (c / g);
    long y0 = ext[2] * (c / g);
    long dx = absB / g;          // x 的通解步长
    long dy = absA / g;          // y 的通解步长

    // 取 x 的最小非负整数解
    // x = x0 + dx * t，t = ceil(-x0/dx)
    long t = (long) Math.ceil((double) (-x0) / dx);
    long xMin = x0 + dx * t;
    long yCorr = y0 - (b > 0 ? dy : -dy) * t;  // 考虑 b 的符号

    // 同时提供 x 最小正整数的 y
    return new long[]{xMin, yCorr, dx, dy};
}
```

### 模线性方程 `ax ≡ b (mod m)` 的完整解法

等价于 `ax - my = b`。调用扩展欧几里得。

```java
/**
 * 求解 ax ≡ b (mod m)
 * 返回 [最小非负解, 步长] 或 null（无解）
 * 通解: x = result[0] + k * result[1], k ∈ ℤ
 */
public static long[] solveCongruence(long a, long b, long m) {
    long[] ext = exgcdIter(a, m);
    long g = ext[0];
    if (b % g != 0) return null;

    long x0 = ext[1] * (b / g) % m;
    if (x0 < 0) x0 += m;
    long step = m / g;
    return new long[]{x0, step};
}

// 示例：解 14x ≡ 30 (mod 100)
// gcd(14, 100) = 2，30 % 2 = 0，有解
// exgcd(14, 100) → -7×14 + 1×100 = 2 → x₀ = -7
// x = -7 × (30/2) = -105 ≡ 95 (mod 50)
// 通解：x ≡ 95 (mod 50)，即 x = 95 + 50k
```

### 题 a) 分数的加减乘除（Fraction Arithmetic）

**问题描述**：实现分数（有理数）的四则运算。任何结果必须化为最简分数，分母非负，结果为假分数时为带分数或假分数均可。

**核心推导**：
- 约分：`约分后的分子 / 约分后的分母 = 原分子/gcd / 原分母/gcd`
- 通分：加减法先通分，`lcm = m₁ / gcd(m₁, m₂) × m₂`
- 乘法：`a₁/b₁ × a₂/b₂ = (a₁×a₂) / (b₁×b₂)`，再约分
- 除法：`a₁/b₁ ÷ a₂/b₂ = (a₁×b₂) / (b₁×a₂)`，再约分

```java
public class Fraction {
    public long num;    // 分子（可正可负）
    public long den;    // 分母（始终为正）

    public Fraction(long num, long den) {
        if (den == 0) throw new ArithmeticException("分母不能为零");
        if (den < 0) { num = -num; den = -den; }
        long g = gcd(Math.abs(num), den);
        this.num = num / g;
        this.den = den / g;
    }

    // ----- 加法 -----
    public Fraction add(Fraction other) {
        long lcm = lcm(this.den, other.den);
        long newNum = this.num * (lcm / this.den) + other.num * (lcm / other.den);
        return new Fraction(newNum, lcm);
    }

    // ----- 减法 -----
    public Fraction subtract(Fraction other) {
        long lcm = lcm(this.den, other.den);
        long newNum = this.num * (lcm / this.den) - other.num * (lcm / other.den);
        return new Fraction(newNum, lcm);
    }

    // ----- 乘法 -----
    public Fraction multiply(Fraction other) {
        return new Fraction(this.num * other.num, this.den * other.den);
    }

    // ----- 除法 -----
    public Fraction divide(Fraction other) {
        if (other.num == 0) throw new ArithmeticException("除数不能为零");
        return new Fraction(this.num * other.den, this.den * other.num);
    }

    // ----- 比较 -----
    public int compareTo(Fraction other) {
        long lcm = lcm(this.den, other.den);
        long left = this.num * (lcm / this.den);
        long right = other.num * (lcm / other.den);
        return Long.compare(left, right);
    }

    @Override
    public String toString() {
        if (den == 1) return String.valueOf(num);
        return num + "/" + den;
    }

    private static long gcd(long a, long b) {
        while (b != 0) { long t = a % b; a = b; b = t; }
        return a;
    }

    private static long lcm(long a, long b) {
        return a / gcd(a, b) * b;
    }
}
```

**复杂度**：O(log min(a,b)) 每次运算，因为 gcd 为瓶颈。
**空间**：O(1)。

**测试用例**：

| 序号 | 运算 | 输入 | 结果 |
|------|------|------|------|
| 1 | 加法 | 1/3 + 1/6 | 1/2 |
| 2 | 减法 | 5/6 - 1/3 | 1/2 |
| 3 | 乘法 | 2/3 × 3/4 | 1/2 |
| 4 | 除法 | 1/2 ÷ 3/4 | 2/3 |
| 5 | 负数 | -1/3 + 1/2 | 1/6 |
| 6 | 整数 | 3/1 + 1/3 | 10/3 |

### 题 b) 最大公约数之和 Σgcd(i, n)

**问题描述**：给定正整数 `n`，求 `S(n) = Σ_{i=1}^{n} gcd(i, n)`。

**朴素算法**：O(n log n)，`n` 大到 10⁶ 即超时。

**核心推导**：

利用性质：`gcd(i, n)` 必然是 `n` 的因子。

```
Σ_{i=1}^{n} gcd(i, n) = Σ_{d|n} d × cnt(d)

其中 cnt(d) = #{i : 1 ≤ i ≤ n, gcd(i, n) = d}
```

关键观察：`gcd(i, n) = d` 等价于 `gcd(i/d, n/d) = 1`，所以：
```
cnt(d) = φ(n/d)  （欧拉函数）
```

因此：
```
S(n) = Σ_{d|n} d × φ(n/d)
```

**优化**：枚举 `n` 的所有因子 `d`（O(√n)），对每个 `d` 计算 `φ(n/d)`。

```java
/**
 * 求 S(n) = Σ_{i=1}^{n} gcd(i, n)
 * 基于公式 S(n) = Σ_{d|n} d × φ(n/d)
 * 时间复杂度 O(√n × log n)
 */
public static long sumGcd(long n) {
    long sum = 0;
    long sqrt = (long) Math.sqrt(n);
    for (long d = 1; d <= sqrt; d++) {
        if (n % d == 0) {
            sum += d * phi(n / d);
            long other = n / d;
            if (other != d) {
                sum += other * phi(d);
            }
        }
    }
    return sum;
}

// 欧拉函数
private static long phi(long n) {
    long result = n;
    long temp = n;
    for (long p = 2; p * p <= temp; p++) {
        if (temp % p == 0) {
            while (temp % p == 0) temp /= p;
            result -= result / p;
        }
    }
    if (temp > 1) result -= result / temp;
    return result;
}
```

**复杂度**：O(√n × √n) = O(n) 最坏（每个因子都需要 φ 计算）。更优做法：先线性筛出 φ 值，再用因子枚举。
**空间**：O(1)。

**测试用例**：

| n | Σgcd(i, n) | 手动验证 |
|---|-----------|---------|
| 1 | 1 | gcd(1,1)=1 |
| 2 | 3 | gcd(1,2)+gcd(2,2)=1+2=3 |
| 3 | 5 | 1+1+3=5 |
| 4 | 8 | 1+2+1+4=8 |
| 5 | 9 | 1+1+1+1+5=9 |
| 6 | 15 | 1+2+3+2+1+6=15 |

### 题 c) 线性丢番图方程——买鸡问题

**问题描述（百鸡百钱）**：公鸡 5 文一只，母鸡 3 文一只，小鸡三只 1 文。用 100 文买 100 只鸡，公鸡、母鸡、小鸡各几只？

**数学建模**：
```
x + y + z = 100      （鸡数）
5x + 3y + z/3 = 100  （钱数）
⇒ 15x + 9y + z = 300
两式相减：14x + 8y = 200 ⇒ 7x + 4y = 100
```

转化为丢番图方程 `7x + 4y = 100`。

```java
/**
 * 百鸡百钱问题
 * 7x + 4y = 100, x ≥ 0, y ≥ 0, z = 100 - x - y ≥ 0
 * 输出所有整数解
 */
public static void buyChicken() {
    // 求解 7x + 4y = 100
    long[] sol = solveDiophantine(7, 4, 100);
    if (sol == null) {
        System.out.println("无解");
        return;
    }

    long x0 = sol[0], y0 = sol[1];
    long dx = sol[2], dy = sol[3]; // dx = 4/g, dy = 7/g

    // 找 x ≥ 0 的最小 t
    // x = x0 + dx * t ≥ 0 → t ≥ ceil(-x0/dx)
    long tMin = (long) Math.ceil((double) (-x0) / dx);

    // 找 y ≥ 0 的最大 t
    // y = y0 - dy * t ≥ 0 → t ≤ floor(y0/dy)
    long tMax = y0 / dy;

    // z = 100 - x - y ≥ 0
    // z = 100 - (x0 + dx*t) - (y0 - dy*t) = 100 - x0 - y0 - (dx - dy)*t
    long z0 = 100 - x0 - y0;
    long dz = dy - dx;  // dx=4, dy=7 → dz=3
    // z ≥ 0: z0 + dz*t ≥ 0 → t ≥ ceil(-z0/dz) if dz>0; t ≤ floor(-z0/dz) if dz<0

    System.out.println("所有非负整数解 (x, y, z) 其中 z = 100-x-y:");
    for (long t = tMin; t <= tMax; t++) {
        long x = x0 + dx * t;
        long y = y0 - dy * t;
        long z = 100 - x - y;
        if (x >= 0 && y >= 0 && z >= 0) {
            System.out.printf("x=%d, y=%d, z=%d%n", x, y, z);
        }
    }
}
```

**输出**：
```
所有非负整数解 (x, y, z) 其中 z = 100-x-y:
x=0, y=25, z=75
x=4, y=18, z=78
x=8, y=11, z=81
x=12, y=4, z=84
```

**复杂度**：O(1) 求解，O(k) 输出所有解（k 为解的个数）。
**拓展**：此题的变体包括找零问题（给定面额找指定金额）、油桶问题（倒水问题）等。

### 题 d) 模线性方程 `ax ≡ b (mod m)`——Pollard-Rho + CRT

**问题描述**：求解 `ax ≡ b (mod m)`，其中 `m` 可能为**大合数**（> 10¹²），需要分解后逐段求解再合并。

**解题策略**：
- **方法 1**：当 `gcd(a, m) = 1` 时，直接求 `a` 的模逆元再乘以 `b`
- **方法 2**：当 `gcd(a, m) > 1` 且 `b % gcd(a, m) ≠ 0` 时，无解
- **方法 3**：当 `gcd(a, m) > 1` 且 `b % gcd(a, m) == 0` 时，可约简为 `(a/g)x ≡ b/g (mod m/g)`，但需注意解的个数为 `g` 个
- **方法 4（合数分解法）**：分解 `m = p₁^e₁ × p₂^e₂ × ...`，逐段求解后 CRT 合并

```java
/**
 * 模线性方程 ax ≡ b (mod m) 的完整解法
 * 返回所有最小非负解（共 gcd(a, m) 个），或空列表（无解）
 */
public static List<Long> solveCongruenceFull(long a, long b, long m) {
    List<Long> solutions = new ArrayList<>();

    long g = gcd(Math.abs(a), m);
    if (b % g != 0) return solutions;  // 无解

    // 约简为 (a/g)x ≡ b/g (mod m/g)
    long aRed = a / g;
    long bRed = b / g;
    long mRed = m / g;

    // 现在 gcd(aRed, mRed) = 1，求逆元
    long inv = modInverse(aRed % mRed, mRed);
    if (inv == -1) return solutions;

    long x0 = (inv * (bRed % mRed)) % mRed;
    if (x0 < 0) x0 += mRed;

    // 共 g 个解：x = x0 + k * mRed, k = 0..g-1
    for (int k = 0; k < g; k++) {
        solutions.add(x0 + k * mRed);
    }

    return solutions;
}

private static long gcd(long a, long b) {
    while (b != 0) { long t = a % b; a = b; b = t; }
    return a;
}

private static long modInverse(long a, long m) {
    long[] ext = exgcdIter(a, m);
    if (ext[0] != 1) return -1;
    return (ext[1] % m + m) % m;
}
```

**测试用例**：

| 方程 | gcd(a,m) | 解数 | 解集 |
|------|----------|------|------|
| 2x ≡ 4 (mod 6) | 2 | 2 | {2, 5} |
| 3x ≡ 5 (mod 7) | 1 | 1 | {4} |
| 4x ≡ 6 (mod 10) | 2 | 2 | {4, 9} |
| 2x ≡ 3 (mod 6) | 2 | 0 | 无解 |
| 6x ≡ 12 (mod 18) | 6 | 6 | {2, 5, 8, 11, 14, 17} |

### 题 e) 模逆元求解——三种方法对比

**问题描述**：求 `x` 使得 `a × x ≡ 1 (mod m)`。

#### 方法 1：扩展欧几里得（通用，要求 gcd(a, m) = 1）

```java
public static long invByExgcd(long a, long m) {
    long[] ext = exgcdIter((a % m + m) % m, m);
    if (ext[0] != 1) return -1;  // 逆元不存在
    return (ext[1] % m + m) % m;
}
```

**适用**：所有 `gcd(a, m) = 1` 的情况，`m` 可以是**合数**。

#### 方法 2：费马小定理（要求 m 为素数）

```
a^(m-1) ≡ 1 (mod m)   （费马小定理）
a × a^(m-2) ≡ 1 (mod m)
⇒ a⁻¹ ≡ a^(m-2) (mod m)
```

```java
/**
 * 费马小定理求逆元，O(log m)
 */
public static long invByFermat(long a, long m) {
    // 需要保证 m 是素数，否则此方法错误
    return modPow(a % m, m - 2, m);
}

private static long modPow(long base, long exp, long mod) {
    long result = 1;
    base %= mod;
    while (exp > 0) {
        if ((exp & 1) == 1) result = (result * base) % mod;
        base = (base * base) % mod;
        exp >>= 1;
    }
    return result;
}
```

**适用**：`m` 为素数（如 10⁹+7），代码简洁，不需要 exgcd。
**不适用**：`m` 为合数。

#### 方法 3：递推预处理逆元（O(n) 预处理，O(1) 查询）

**公式**（`m` 为素数）：
```
inv[1] = 1
inv[i] = m - (m/i) × inv[m % i] % m
```

**推导**：对质数 `p`，设 `k = ⌊p/i⌋`，`r = p % i`，则：
```
p = k × i + r
k × i + r ≡ 0 (mod p)
k × r⁻¹ + i⁻¹ ≡ 0 (mod p)
i⁻¹ ≡ -k × r⁻¹ (mod p)
i⁻¹ ≡ p - (p/i) × inv[p % i] % p
```

```java
/**
 * 预处理 1..n 的逆元
 * n 至少到 m-1
 */
public static long[] precomputeInverse(int n, long m) {
    long[] inv = new long[n + 1];
    inv[1] = 1;
    for (int i = 2; i <= n; i++) {
        inv[i] = m - (m / i) * inv[(int)(m % i)] % m;
    }
    return inv;
}
```

**适用**：需要频繁求逆元（如组合数计算 `C(n,k) mod p`），`m` 为素数。

#### 方法对比

| 指标 | 扩展欧几里得 | 费马小定理 | 递推预处理 |
|------|-------------|-----------|-----------|
| 时间复杂度 | O(log m) | O(log m) | O(n) 预处理，O(1) 查询 |
| m 的要求 | gcd(a,m)=1 | m 为素数 | m 为素数 |
| 单次查询 | 快 | 快 | 极快 |
| 已知条件 | 无 | m 为素数 | m 为素数 |

### 题 f) 最小公倍数——防止溢出的计算技巧

**问题描述**：求多个数的最小公倍数（LCM），正确处理数据溢出。

**核心公式**：
```
lcm(a, b) = a / gcd(a, b) × b    （先除后乘，防止溢出）
```

对于多个数：`lcm(a₁, a₂, ..., aₙ)`：
```
lcm(a₁, a₂, a₃) = lcm(lcm(a₁, a₂), a₃)
```

```java
/**
 * 求两个数的 LCM，先除后乘防止溢出
 */
public static long lcm(long a, long b) {
    if (a == 0 || b == 0) return 0;
    return a / gcd(a, b) * b;
}

/**
 * 求数组的 LCM
 * 如果结果超过 Long.MAX_VALUE，返回 -1 表示溢出
 */
public static long lcmArray(long[] arr) {
    long result = 1;
    for (long num : arr) {
        long g = gcd(result, num);
        // 检查溢出：result / g > Long.MAX_VALUE / num
        if (result / g > Long.MAX_VALUE / num) {
            return -1;  // 溢出
        }
        result = result / g * num;
    }
    return result;
}

/**
 * 使用 BigInteger 避免溢出
 */
public static BigInteger lcmArraySafe(long[] arr) {
    BigInteger result = BigInteger.ONE;
    for (long num : arr) {
        BigInteger biNum = BigInteger.valueOf(num);
        result = result.divide(result.gcd(biNum)).multiply(biNum);
    }
    return result;
}

private static long gcd(long a, long b) {
    while (b != 0) { long t = a % b; a = b; b = t; }
    return a;
}
```

**溢出分析**：
- 对于 64 位 `long`，`lcm(10¹⁰, 10¹¹) = 10²¹` 已溢出（`Long.MAX_VALUE ≈ 9.22×10¹⁸`）
- 错误的写法：`a * b / gcd(a, b)` 在 `a*b` 时已溢出
- 正确的写法：`a / gcd(a, b) * b`
- 即使先除后乘，多个数的 LCM 也可能超过 `Long.MAX_VALUE`，此时需用 `BigInteger`

**测试用例**：

| 输入 | 结果 | 说明 |
|------|------|------|
| lcm(6, 8) | 24 | |
| lcm(12, 18) | 36 | |
| lcm(10⁹, 10⁹+7) | 1000000007000000000 | 仍在 long 范围 |
| lcm(999999937, 999999937) | 999999937 | 素数，略小于 10⁹ |
| lcm(10¹⁰, 10¹¹) | 溢出 | 需 BigInteger |

**LCM 与分数运算的关系**：
```
通分：1/a + 1/b = (b + a) / lcm(a,b)
        = (b + a) / (a / gcd(a,b) × b)
        = (b + a) × gcd(a,b) / (a × b)
```

### 题 g) 同余方程组与中国剩余定理

**问题描述**：解同余方程组
```
x ≡ a₁ (mod m₁)
x ≡ a₂ (mod m₂)
...
x ≡ aₙ (mod mₙ)
```

**两种情形**：

#### 情形 1：模数互质——中国剩余定理（CRT）

若 `gcd(mᵢ, mⱼ) = 1` 对所有 `i ≠ j` 成立，则方程组在模 `M = Πmᵢ` 下有唯一解。

```
x = (Σ aᵢ × Mᵢ × inv(Mᵢ, mᵢ)) mod M

其中 Mᵢ = M / mᵢ，inv(Mᵢ, mᵢ) 是 Mᵢ 模 mᵢ 的逆元
```

```java
/**
 * 中国剩余定理（模数互质）
 * @param a 余数数组
 * @param m 模数数组（两两互质）
 * @return [解, M] 其中 M = Πmᵢ
 */
public static long[] crt(long[] a, long[] m) {
    long M = 1;
    for (long mod : m) M *= mod;

    long x = 0;
    for (int i = 0; i < a.length; i++) {
        long Mi = M / m[i];
        long inv = modInverse(Mi % m[i], m[i]);
        if (inv == -1) return null;
        x = (x + a[i] * Mi % M * inv) % M;
    }
    return new long[]{x, M};
}
```

**复杂度**：O(n log M)。
**适用场景**：前提条件是模数两两互质，需通过质因数分解验证。

#### 情形 2：模数不互质——两两合并法（通用解法）

```java
/**
 * 合并两个同余方程，适用于任意模数（不要求互质）
 * x ≡ a₁ (mod m₁)
 * x ≡ a₂ (mod m₂)
 * 返回 [新余数, 新模数] 或 null（无解）
 */
public static long[] mergeCongruence(long a1, long m1, long a2, long m2) {
    long[] ext = exgcdIter(m1, m2);
    long g = ext[0], p = ext[1];
    long diff = a2 - a1;

    // 检查解的存在性
    if (diff % g != 0) return null;

    // 合并
    long step = m2 / g;
    long t = (diff / g * p) % step;
    if (t < 0) t += step;

    long M = m1 / g * m2;  // lcm(m1, m2)
    long x = (a1 + m1 * t) % M;
    if (x < 0) x += M;

    return new long[]{x, M};
}

/**
 * 解任意同余方程组（通用）
 * @param a 余数数组
 * @param m 模数数组
 * @return [解, 模数] 或 null（无解）
 */
public static long[] solveCongruenceSystem(long[] a, long[] m) {
    long x = a[0], mod = m[0];
    for (int i = 1; i < a.length; i++) {
        long[] result = mergeCongruence(x, mod, a[i], m[i]);
        if (result == null) return null;
        x = result[0];
        mod = result[1];
    }
    return new long[]{x, mod};
}
```

**复杂度**：O(n log M)，每步合并需要一次 exgcd。
**测试用例**：

| 方程组 | 解 | 说明 |
|--------|------|------|
| x ≡ 2 (mod 3), x ≡ 3 (mod 5), x ≡ 2 (mod 7) | 23 (mod 105) | 经典"物不知数" |
| x ≡ 1 (mod 4), x ≡ 3 (mod 6) | 9 (mod 12) | 模数不互质 |
| x ≡ 1 (mod 2), x ≡ 2 (mod 4) | 无解 | 1 奇 2 偶矛盾 |
| x ≡ 3 (mod 8), x ≡ 7 (mod 12) | 31 (mod 24) | 即 7 (mod 24) |

### 题 h) RSA 密钥生成中的 d 值计算

**问题描述**：RSA 密钥生成的最后一步，已知公钥指数 `e` 和欧拉函数值 `φ(n) = (p-1)(q-1)`，求私钥指数 `d` 满足：

```
e × d ≡ 1 (mod φ(n))
```

这正是求 `e` 模 `φ(n)` 的逆元，用扩展欧几里得一步求解。

#### 完整 RSA 密钥生成流程

```java
import java.math.BigInteger;
import java.security.SecureRandom;

public class RSAKeyGen {
    private BigInteger p, q, n, phi, e, d;
    private SecureRandom random;

    public RSAKeyGen(int bitLength) {
        random = new SecureRandom();
        // 1. 随机选择两个大素数 p, q
        p = BigInteger.probablePrime(bitLength / 2, random);
        q = BigInteger.probablePrime(bitLength / 2, random);

        // 2. 计算 n = p × q
        n = p.multiply(q);

        // 3. 计算 φ(n) = (p-1)(q-1)
        phi = p.subtract(BigInteger.ONE)
               .multiply(q.subtract(BigInteger.ONE));

        // 4. 选择公钥指数 e：1 < e < φ(n) 且 gcd(e, φ(n)) = 1
        //    通常用 65537 (2^16 + 1)
        e = BigInteger.valueOf(65537);

        // 5. 计算私钥 d：e × d ≡ 1 (mod φ(n))
        //    这是扩展欧几里得算法
        d = e.modInverse(phi);

        // 验证：e × d mod φ(n) = 1
        assert e.multiply(d).mod(phi).equals(BigInteger.ONE);
    }

    // 加密：c = m^e mod n
    public BigInteger encrypt(BigInteger m) {
        return m.modPow(e, n);
    }

    // 解密：m = c^d mod n
    public BigInteger decrypt(BigInteger c) {
        return c.modPow(d, n);
    }

    public void printKeys() {
        System.out.println("公钥 (n, e): (" + n + ", " + e + ")");
        System.out.println("私钥 (n, d): (" + n + ", " + d + ")");
        System.out.println("p = " + p + "\nq = " + q);
        System.out.println("φ(n) = " + phi);
    }
}
```

**核心步骤 5 的手动执行**（举例 `p=11, q=13, n=143, φ=120, e=7`）：

方程：`7d ≡ 1 (mod 120)`

用扩展欧几里得求解 `7x + 120y = 1`：

```
exgcd(7, 120):
  120 = 17×7 + 1
  7 = 7×1 + 0

exgcd(120, 7) → exgcd(7, 1) → exgcd(1, 0)
  → 返回 (1, 1, 0)

回溯：
exgcd(7, 1): x = 0, y = 1 - 7×0 = 1    → (1, 0, 1)
exgcd(120, 7): x = 1, y = 0 - 17×1 = -17 → (1, 1, -17)

所以 7×(-17) + 120×1 = 1
⇒ 7 × (-17) ≡ 1 (mod 120)
⇒ d ≡ -17 (mod 120) ≡ 103
```

验证：`7 × 103 = 721 = 6×120 + 1 ≡ 1 (mod 120) ✓`

**扩展欧几里得在 RSA 中的关键地位**：
- 选择 `e` 时：需验证 `gcd(e, φ(n)) = 1`（互质判定）
- 计算 `d` 时：求模逆元 `d ≡ e⁻¹ mod φ(n)`
- **没有扩展欧几里得，RSA 无法生成私钥**——这是该算法在密码学中最重要的应用之一

#### 模板 1：最小非负整数特解

```java
// 求 ax + by = c 的一组特解，x 最小非负
public static long[] solveDiophantineMinX(long a, long b, long c, boolean absFlag) {
    long absA = Math.abs(a), absB = Math.abs(b);
    long[] ext = exgcdIter(absA, absB);
    long g = ext[0];
    if (c % g != 0) return new long[]{0, 0, 0, 0};
    long x0 = ext[1] * (c / g);
    if (a < 0) x0 = -x0;
    long dx = absB / g;
    long t = (long) Math.ceil((double) (-x0) / dx);
    long x = x0 + dx * t;
    return new long[]{x, 0, dx, g};
}
```

#### 模板 2：ax ≡ b (mod m) 的最小非负解

```java
public static long solveLinearMin(long a, long b, long m) {
    long[] ext = exgcdIter((a % m + m) % m, m);
    long g = ext[0];
    if (b % g != 0) return -1;
    long x0 = ext[1] * (b / g) % m;
    return (x0 % m + m) % m;
}
```

#### 模板 3：大数求逆元

```java
public static BigInteger modInverse(BigInteger a, BigInteger m) {
    // 不调用 BigInteger.modInverse 自己实现
    BigInteger[] ext = exgcdBigInt(a.mod(m), m);
    if (!ext[0].equals(BigInteger.ONE)) return null;
    return ext[1].mod(m);
}

private static BigInteger[] exgcdBigInt(BigInteger a, BigInteger b) {
    if (b.equals(BigInteger.ZERO))
        return new BigInteger[]{a, BigInteger.ONE, BigInteger.ZERO};
    BigInteger[] res = exgcdBigInt(b, a.mod(b));
    BigInteger x1 = res[1], y1 = res[2];
    return new BigInteger[]{res[0], y1, x1.subtract(a.divide(b).multiply(y1))};
}
```
