# 词向量应用场景详解

## 前置概念：什么是词向量

**词向量（Word Embedding / Word Vector）** 是将自然语言中的词语映射为稠密实数向量的技术。与传统 One-Hot 编码相比，词向量具有以下特征：

| 特性 | One-Hot 编码 | 词向量（分布式表示） |
|------|-------------|-------------------|
| 向量长度 | 等于词典大小（数万维） | 固定低维（通常 50~300 维） |
| 稀疏性 | 极度稀疏（仅 1 个 1，其余为 0） | 稠密（每个维度均为浮点数） |
| 语义关系 | 无法体现（"猫"与"狗"余弦为 0） | 语义相近的词向量也相近 |
| 计算效率 | 高维稀疏，存储与计算开销大 | 低维稠密，适合神经网络处理 |

**代表性模型：**

| 模型 | 年份 | 核心特点 |
|------|------|---------|
| Word2Vec（Mikolov et al.） | 2013 | CBOW / Skip-gram，效率高，应用最广 |
| GloVe（Pennington et al.） | 2014 | 全局词共现矩阵分解，利用统计信息 |
| FastText（Facebook） | 2016 | 引入子词（subword）信息，可处理 OOV 词 |
| BERT Embeddings（Google） | 2018 | 上下文相关的动态词向量 |

> 词向量的核心假设：**"语义相似的词出现在相似的上下文中"**（Distributional Hypothesis, Harris 1954）

## 应用一：单词语义相关性

### 核心原理

利用词向量的**余弦相似度（Cosine Similarity）** 来度量词语间的语义接近程度：

$$
\text{cosine\_similarity}(\mathbf{v}_a, \mathbf{v}_b) = \frac{\mathbf{v}_a \cdot \mathbf{v}_b}{\|\mathbf{v}_a\| \cdot \|\mathbf{v}_b\|}
$$

值域 $[-1, 1]$，越接近 1 表示语义越相似。

### 子场景 A：查找近义词

给定词语 $w$，找出词典中与其余弦相似度最大的 $N$ 个词语。

```python
import numpy as np

def find_nearest_words(embedding, word, vocab, top_n=10):
    """
    embedding: dict[str, np.ndarray] — 词向量字典
    word: 目标词语
    vocab: 词典列表
    top_n: 返回前 N 个近义词
    """
    if word not in embedding:
        return []

    vec = embedding[word]
    similarities = []

    for w in vocab:
        if w == word:
            continue
        v = embedding[w]
        cos_sim = np.dot(vec, v) / (np.linalg.norm(vec) * np.linalg.norm(v))
        similarities.append((w, cos_sim))

    similarities.sort(key=lambda x: x[1], reverse=True)
    return similarities[:top_n]
```

### 子场景 B：语义相似度比较

直接计算两个词向量的余弦值。

```python
def word_similarity(embedding, word_a, word_b):
    """返回两个词语的语义相似度"""
    va, vb = embedding[word_a], embedding[word_b]
    return np.dot(va, vb) / (np.linalg.norm(va) * np.linalg.norm(vb))
```

### 实际应用

| 领域 | 示例 |
|------|------|
| 搜索引擎 | 用户搜"汽车" → 召回含"轿车"、"车辆"的页面（同义词扩展） |
| 文本纠错 | 检测"苹果" → 根据上下文判断是水果还是公司，推荐最相关替代 |
| 同义词词典构建 | 自动挖掘大规模同义词 / 近义词库 |
| 推荐系统 | 基于用户历史搜索词的语义相似度推荐相关内容 |

## 应用二：词语类比推理

### 核心原理

词向量最著名的特性之一是支持**向量算术（Vector Arithmetic）**。经典案例：

$$
\text{vector}(\text{king}) - \text{vector}(\text{man}) + \text{vector}(\text{woman}) \approx \text{vector}(\text{queen})
$$

**推导逻辑：**
- $\text{king} - \text{man}$ 提取出"王室"语义分量
- 再加上 $\text{woman}$ 赋予"女性"语义
- 结果应最接近 $\text{queen}$

### 代码实现

```python
def analogy(embedding, word_a, word_b, word_c, vocab, top_n=5):
    """
    寻找 word_a:word_b = word_c:word_d 中的 word_d

    计算: embedding[word_b] - embedding[word_a] + embedding[word_c]
    找到结果最近邻的词
    """
    target_vec = embedding[word_b] - embedding[word_a] + embedding[word_c]
    candidates = []

    for w in vocab:
        if w in {word_a, word_b, word_c}:
            continue
        v = embedding[w]
        cos_sim = np.dot(target_vec, v) / (np.linalg.norm(target_vec) * np.linalg.norm(v))
        candidates.append((w, cos_sim))

    candidates.sort(key=lambda x: x[1], reverse=True)
    return candidates[:top_n]


# 使用示例：
# analogy(embedding, "巴黎", "法国", "东京", vocab)
# → [("日本", 0.82), ("京都", 0.73), ...]
```

### 典型类比案例

| 关系类型 | 示例 |
|---------|------|
| 国家—首都 | 法国 : 巴黎 :: 日本 : 东京 |
| 时态 | 走 : 走了 :: 吃 : 吃了 |
| 性别 | 男人 : 女人 :: 国王 : 女王 |
| 比较级 | 好 : 更好 :: 大 : 更大 |
| 整体—部分 | 手 : 手指 :: 脚 : 脚趾 |

### 实际应用

| 场景 | 说明 |
|------|------|
| 知识图谱补全 | 推断未直接建模的实体间关系 |
| 智能问答 | 理解"X 和 Y 的关系类似于 Z 和什么"这类推理问题 |
| 跨语言迁移 | 跨语言词向量对齐后可做零样本翻译推理 |

## 应用三：短文本相似度

### 核心原理

将短文本中的每个词向量求平均（或加权平均），得到**句子嵌入（Sentence Embedding）**，再计算句子嵌入的余弦相似度：

$$
\text{sentence\_vec}(S) = \frac{1}{|S|} \sum_{w \in S} \mathbf{v}_w
$$

这是**词袋模型（Bag-of-Words）** 思想的向量化延续——忽略词序，仅考虑词义分布。

### 代码实现

```python
def sentence_similarity(embedding, sent_a, sent_b):
    """
    计算两段短文本的语义相似度
    sent_a, sent_b: 分词后的词语列表
    """
    def sentence_vec(tokens):
        vecs = [embedding[w] for w in tokens if w in embedding]
        if not vecs:
            return np.zeros(300)
        return np.mean(vecs, axis=0)

    va = sentence_vec(sent_a)
    vb = sentence_vec(sent_b)

    norm_product = np.linalg.norm(va) * np.linalg.norm(vb)
    if norm_product == 0:
        return 0.0
    return float(np.dot(va, vb) / norm_product)
```

### 局限性与改进

| 问题 | 说明 | 改进方案 |
|------|------|---------|
| 忽略词序 | "猫追狗"与"狗追猫"的向量相同 | TF-IDF 加权、Bi-LSTM / BERT 编码器 |
| OOV 词 | 未登录词无法获得向量 | FastText（子词级）、BPE 分词 |
| 语境缺失 | 一词多义（"苹果"→水果/公司）无法区分 | ELMo / BERT 等上下文感知模型 |
| 短文本长度敏感 | 过短时均值受 OOV 影响大 | 引入字符级嵌入或 n-gram 特征 |

### 实际应用

- **智能客服 FAQ 匹配**：用户提问 → 与标准问题库做语义相似度匹配 → 返回最接近的标准答案
- **候选句去重**：文本摘要、信息检索中对语义重复的候选结果去重
- **聚类与分类**：短文本转向量后用于 K-Means 聚类或 SVM 分类
- **检索式对话系统**：在候选回复集中匹配最相关的回复

## 应用四：文本分类

### 核心原理

将文档中所有词向量汇聚作为文档的特征表示，输入分类器：

```
文档 -分词→ 词序列 -查向量→ [v₁, v₂, ..., vₙ] -池化→ doc_vec -分类器→ 类别标签
```

### 代码实现

```python
from sklearn.linear_model import LogisticRegression

def doc_vector(embedding, tokens):
    """将文档转为固定维度的向量表示"""
    vecs = [embedding[w] for w in tokens if w in embedding]
    if not vecs:
        return np.zeros(300)
    return np.mean(vecs, axis=0)

# 训练
X_train = np.array([doc_vector(emb, doc) for doc in train_docs])
y_train = train_labels
clf = LogisticRegression(max_iter=1000)
clf.fit(X_train, y_train)

# 预测
X_test = np.array([doc_vector(emb, doc) for doc in test_docs])
predictions = clf.predict(X_test)
```

### 实际应用

| 任务 | 说明 |
|------|------|
| 情感分析 | 评论 / 推文 → 正面 / 负面 / 中性 |
| 垃圾邮件检测 | 邮件内容向量化 → 二分类 |
| 新闻分类 | 体育 / 政治 / 科技 等多分类 |
| 意图识别 | 对话系统中识别用户查询意图 |

## 应用五：序列标注与命名实体识别（NER）

### 核心原理

词向量作为神经网络的**输入特征**，与字符特征、CRF 层结合处理序列标注任务：

```text
输入：     "昨天  我  去  了   北京"
词向量：    [v₁]  [v₂] [v₃] [v₄] [v₅]
                 ↓
          Bi-LSTM / Transformer 编码器
                 ↓
          CRF 解码层
                 ↓
输出：     O    O   O   O   B-LOC
```

### 实际应用

- **命名实体识别**：识别人名、地名、机构名
- **词性标注（POS Tagging）**：标注每个词的词性（名词 / 动词 / ...）
- **语义角色标注**：标注句子中"谁对谁做了什么"
- **医学文本**：从电子病历中抽取症状、药物、检查等实体

## 应用六：机器翻译

### 核心原理

词向量在编码器—解码器架构中作为**输入与桥梁表示**：

- **编码器**：源语言词序列 → 词向量 → RNN/Transformer → 上下文表示
- **解码器**：上下文表示 → 生成目标语言词序列

### 跨语言词向量对齐

在同一向量空间中对齐双语词向量，实现**跨语言迁移**：

```text
中文词向量空间          英文词向量空间
    v(猫)                       v(cat)
    v(狗)                       v(dog)
         ↘                   ↙
         对齐后的联合表示空间
```

## 应用七：推荐系统

### 核心原理

将用户 / 物品的交互序列视为"词序列"，用 Word2Vec 的 Skip-gram 思想学习表示：

| NLP 概念 | 推荐系统映射 |
|---------|-------------|
| 语料库 | 用户行为序列集合 |
| 词 | 物品（商品 / 视频 / 文章） |
| 上下文窗口 | 相邻的浏览 / 购买序列 |
| 相似词 | 用户行为上相关的物品 |

### 实际案例

- **Airbnb**：房源向量 → 相似房源推荐 + 搜索排序
- **YouTube**：观看历史作"词序列" → 视频向量 → 相关推荐
- **淘宝**：用户点击序列的 Item2Vec → 商品相似度搜索

## 应用八：异常检测与离群词发现

### 核心原理

通过分析词向量在空间中的分布密度来判断语义异常：

```python
def outlier_detection(embedding, words):
    """从一组词语中检测语义异常的离群词"""
    vecs = np.array([embedding[w] for w in words])
    center = np.mean(vecs, axis=0)

    distances = []
    for i, w in enumerate(words):
        dist = np.linalg.norm(vecs[i] - center)
        distances.append((w, dist))

    distances.sort(key=lambda x: x[1], reverse=True)
    return distances
```

### 实际应用

- **测试集质量检查**：在一组选项中选择语义最无关的"干扰项"
- **数据清洗**：检测文本语料中的格式错误或乱码词
- **日志异常检测**：系统日志序列中语义跳跃大的片段可能是异常

## 词向量适用性对比一览

| 应用场景 | 核心操作 | 是否需要标注数据 | 典型模型 |
|---------|---------|----------------|---------|
| 近义词查找 | 余弦相似度 Top-N | 无需 | Word2Vec / GloVe |
| 词语类比 | 向量算术 + 近邻 | 无需 | Word2Vec |
| 短文本相似度 | 均值池化 + 余弦 | 无需 | Word2Vec / BERT |
| 文本分类 | 池化 + 分类器 | 需要标签 | Word2Vec + LR/CNN |
| NER | 序列标注网络 | 需要标签 | BERT / Bi-LSTM+CRF |
| 机器翻译 | Seq2Seq 编码器 | 需要平行语料 | Transformer |
| 推荐系统 | Item2Vec | 无需（自监督） | Word2Vec skip-gram |
| 异常检测 | 空间距离分析 | 无需 | 任意词向量 |
