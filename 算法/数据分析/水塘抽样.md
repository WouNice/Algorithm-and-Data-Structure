# 水塘抽样（Reservoir Sampling）

## 应用场景

从流式数据的 $n$ 个元素中，随机抽取 $k$ 个样本（$k \leq n$），需满足：

- **等概率性**：每个元素被选中的概率相同；
- **一次遍历**：时间复杂度不超过 $O(n)$，空间复杂度不超过 $O(k)$。

## 算法步骤

1. 初始化长度为 $k$ 的缓存列表 $L$；
2. 遍历第 $i$ 个元素（从 $1$ 开始计数）：
   - 若 $i \leq k$，则将该元素直接存入缓存列表；
   - 若 $i > k$，则以 $\dfrac{k}{i}$ 的概率保留该元素：生成随机数 $t \sim U(0, 1)$，若 $t < \dfrac{k}{i}$，则将该元素替换入缓存列表，并从列表中随机丢弃一个原有元素（每个原有元素被丢弃的概率为 $\dfrac{1}{i}$）；
3. 遍历结束后，缓存列表 $L$ 即为抽取的样本。

## 算法证明

### 等概率性

用数学归纳法证明。

**基础情况**：$n = k$ 时，所有元素必然被选中，概率均为 $1$，结论成立。

**归纳步骤**：假设 $n = m$（$m \geq k$）时结论成立，即前 $m$ 个元素被选中的概率均为 $\dfrac{k}{m}$。需证 $n = m + 1$ 时结论仍成立。

第 $m + 1$ 个元素被保留的概率为：

$$
P_{m+1}(m+1) = \frac{k}{m+1}
$$

遍历第 $m + 1$ 个元素时，有 $\dfrac{k}{m+1}$ 的概率触发替换操作。缓存中原有的 $k$ 个样本中，每个被丢弃的概率为：

$$
\frac{k}{m+1} \times \frac{1}{k} = \frac{1}{m+1}
$$

因此，前 $m$ 个元素在遍历第 $m + 1$ 个元素之后仍被保留的概率为：

$$
P_{m+1}(i) = P_m(i) \times \left(1 - \frac{1}{m+1}\right)
          = \frac{k}{m} \times \frac{m}{m+1}
          = \frac{k}{m+1} \quad (i = 1, 2, \dots, m)
$$

与第 $m + 1$ 个元素被选中的概率 $\dfrac{k}{m+1}$ 相同。归纳成立，得证。

### 复杂度分析

- **时间复杂度**：$O(n)$，只需遍历一次所有元素；
- **空间复杂度**：$O(k)$，仅需缓存 $k$ 个样本。

## 代码实现

> 代码遵循 PEP 8、PEP 484 及 pylint 规范，使用 NumPy 风格的文档注释。

```python
import random
from typing import Any, Iterator, List


def reservoir_sampling(population: Iterator[Any], n_sample: int) -> List[Any]:
    """水塘抽样（Reservoir Sampling）

    Parameters
    ----------
    population : Iterator[Any]
        所有元素的可迭代对象（流式数据）
    n_sample : int
        需要抽取的样本数

    Returns
    -------
    sample : List[Any]
        选中的样本列表
    """
    cache: List[Any] = []
    i = 1  # 1-indexed，与算法描述一致
    for elem in population:
        if i <= n_sample:
            cache.append(elem)
        elif random.random() < n_sample / i:
            cache[random.randint(0, n_sample - 1)] = elem
        i += 1
    return cache
```

测试用例：

```python
# 样本数等于总数，全部元素被选中
reservoir_sampling(range(5), 5)  # [0, 1, 2, 3, 4]

# 从 1000 个元素中抽取 5 个（每次结果随机）
reservoir_sampling(range(1000), 5)
```
