# 如何在一个无序数据中合理抽样？

## 问题

给定一个有 `m` 个元素的无序数组，需要抽取 `n` 个样本，怎样合理抽样？

> **关键认知**：无序数据 ≠ 随机数据。无序数组只是**未排序**，但数据内部可能存在隐藏的顺序或规律（如按时间写入、按类别聚集等），因此直接取前 `n` 个元素作为样本会导致**系统性偏差**（sampling bias）。

## 基础方案：随机抽样

### 方案一：直接随机抽样

从 `m` 个元素中均匀随机选取 `n` 个，不依赖数组的初始顺序。

```python
import random

def random_sample(arr, n):
    """从数组中随机抽取 n 个不重复元素"""
    return random.sample(arr, n)
```

- **时间复杂度**：O(m)（需遍历或构建索引）
- **空间复杂度**：O(n)
- **优点**：简单直接，无需修改原数组
- **缺点**：多次抽样时每次都要重新随机选取

### 方案二：洗牌后截取

先用洗牌算法将数组打乱成随机排列，再取前 `n` 个。

```python
import random

def shuffle_and_take(arr, n):
    """Fisher-Yates 洗牌后取前 n 个"""
    shuffled = arr[:]                   # 副本，不修改原数组
    for i in range(len(shuffled) - 1, -1, -1):
        j = random.randint(0, i)
        shuffled[i], shuffled[j] = shuffled[j], shuffled[i]
    return shuffled[:n]
```

- **时间复杂度**：O(m)
- **空间复杂度**：O(m)（需要副本）
- **优点**：洗牌后后续抽样极快
- **缺点**：全量洗牌开销较大

### 方案对比

| 方案 | 单次抽样耗时 | 多次抽样耗时 | 空间开销 | 是否修改原数据 |
|------|------------|------------|---------|--------------|
| 直接随机抽样 | O(m) | 每次 O(m) | O(n) | 否 |
| 洗牌后截取 | O(m) | 首次 O(m)，后续 O(1) | O(m) | 是（可传副本） |
| 部分 Fisher-Yates | O(n) | 每次 O(n) | O(n) | 是 |

> **部分 Fisher-Yates**：只需洗牌前 `n` 个位置即可，不必遍历整个数组，单次抽样可优化到 O(n)。

## 延伸问题

### 延伸 1：未知总数（数据流场景）→ 水塘抽样

**场景**：无序数组存储在文件中，元素总数未知，无法一次性读入内存。

**方法**：[水塘抽样（Reservoir Sampling）](https://en.wikipedia.org/wiki/Reservoir_sampling)

```python
import random

def reservoir_sample(stream, n):
    """从数据流中均匀抽取 n 个样本（水塘抽样）"""
    reservoir = []
    for i, item in enumerate(stream):
        if i < n:
            reservoir.append(item)          # 初始化水塘
        else:
            j = random.randint(0, i)        # 随机决定是否替换
            if j < n:
                reservoir[j] = item
    return reservoir
```

**算法原理**（Algorithm R）：

1. 前 `n` 个元素直接放入"水塘"（reservoir）
2. 第 `i` 个元素（i ≥ n）以 `n / (i+1)` 的概率替换水塘中的一个元素
3. 遍历结束时，水塘中的元素是均匀抽样

**复杂度**：

| 维度 | 值 |
|------|-----|
| 时间复杂度 | O(m) |
| 空间复杂度 | O(n) |
| 是否需预知总数 | 否 |
| 是否支持流式 | 是 |

> **优化变体**：当 n 远小于 m 时，可用跳跃抽样（skip-based reservoir sampling）跳过大量无需处理的元素，性能更高。

### 延伸 2：多次抽样 → 一次洗牌 + 滑动窗口

**场景**：后续需要多次从同一数据集中抽样。

**策略**：先做一次全量洗牌得到随机序列，后续每次抽样只需指定一个随机的起始位置，向后取连续 `n` 个元素。

```python
import random

class MultipleSampler:
    def __init__(self, arr):
        self.shuffled = arr[:]
        # 一次性 Fisher-Yates 洗牌
        for i in range(len(self.shuffled) - 1, 0, -1):
            j = random.randint(0, i)
            self.shuffled[i], self.shuffled[j] = self.shuffled[j], self.shuffled[i]
        self.size = len(self.shuffled)

    def sample(self, n):
        """每次 O(1) 获取 n 个样本"""
        start = random.randint(0, self.size - 1)
        if start + n <= self.size:
            return self.shuffled[start:start + n]
        # 环形处理
        return self.shuffled[start:] + self.shuffled[:start + n - self.size]
```

**各阶段开销**：

| 阶段 | 复杂度 |
|------|--------|
| 初始化（一次洗牌） | O(m) |
| 每次抽样 | O(n) |
| 多次抽样均摊 | O(m + k·n)，k 为抽样次数 |

### 延伸 3：高性能洗牌 → Fisher-Yates

Fisher-Yates（也称 Knuth shuffle）是目前最优的均匀洗牌算法，保证每个排列等概率出现。

```python
def fisher_yates(arr):
    """原地 Fisher-Yates 洗牌，时间复杂度 O(n)，空间复杂度 O(1)"""
    for i in range(len(arr) - 1, 0, -1):
        j = random.randint(0, i)   # 含 i 的均匀随机整数
        arr[i], arr[j] = arr[j], arr[i]
```

**关键特性**：

| 特性 | 说明 |
|------|------|
| **正确性** | 每个排列的概率均为 1/m!，严格均匀 |
| **时间复杂度** | O(m)，取决于 `random.randint` 的实现 |
| **空间复杂度** | O(1)（原地交换） |
| **是否稳定** | 否（排序不稳定是） |

> **注意**：`random.randint(0, i)` 是闭区间 [0, i]。若不慎写成 `randint(0, i-1)`，则无法保证均匀性。

## 总结：如何选择？

| 场景 | 推荐方案 |
|------|---------|
| 单次抽样，内存充足 | `random.sample()` |
| 单次抽样，数据流/文件 | 水塘抽样（Reservoir Sampling） |
| 多次抽样 | 一次 Fisher-Yates 洗牌 + 滑动窗口 |
| 数据量大，需要部分抽样 | 部分 Fisher-Yates（只洗前 n 个） |
| 需严格均匀分布 | Fisher-Yates（而非简单随机交换 `random.shuffle` 以外的自制实现） |
